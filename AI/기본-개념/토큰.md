# 토큰 (Token)
> 최종 업데이트: 2026-05-02 | 기준 정보: GPT-4o, Claude 4.x, Gemini 1.5 시점

## 개념
토큰은 **LLM이 텍스트를 인식·처리하는 최소 단위**. 단어도 아니고 글자도 아닌, 그 중간쯤 되는 "조각"이다.

쉽게 비유하면, 사람이 책을 글자 하나하나가 아니라 자주 쓰는 단어/구 단위로 읽듯이, AI도 자주 등장하는 패턴을 묶어 토큰 단위로 읽는다. 모델은 미리 정의된 **사전(vocabulary)** 의 토큰을 기본 단위로 인식하지만, 사전에 없는 단어는 더 작은 조각(서브워드/바이트) 으로 분해해서 처리하므로 어떤 입력이든 토큰화 자체는 항상 성공한다.

## 배경/역사
- **1994년**: BPE(Byte Pair Encoding) 알고리즘이 데이터 압축 용도로 처음 제안됨
- **2015년**: 신경망 기계번역(NMT)에 BPE 적용 (Sennrich et al.) — 미등록 단어(OOV) 문제 해결
- **2018년**: GPT-2가 BPE를 채용. vocab 약 50K
- **2022년~**: GPT-3.5/4가 `cl100k_base` (vocab 약 100K)로 다국어 처리 개선
- **2024년~**: 멀티모달 토큰화 등장 — 이미지·오디오도 토큰으로 변환되어 동일 컨텍스트에 포함

## 왜 토큰으로 쪼개는가? (Tokenization)
- 모델은 학습 시점에 만들어진 **고정된 사전**(보통 5만~20만개)을 기준으로 입력을 인식
- 자주 등장하는 글자 묶음일수록 하나의 토큰으로 묶임 → 효율적
- 처음 보는 단어는 더 작은 조각으로 쪼개서 표현 (예: `chatGPT` → `chat` + `GPT`)
- **바이트 단위까지 fallback** 가능 → 사전에 없는 단어, 신조어, 이모지, 특수문자도 모두 토큰화됨

대표 알고리즘:

| 알고리즘 | 사용 모델 | 특징 |
|----------|-----------|------|
| BPE (Byte Pair Encoding) | GPT 시리즈, Claude | 자주 쓰이는 바이트 쌍을 반복 병합 |
| WordPiece | BERT 계열 | BPE와 유사, 확률 기반 병합 |
| SentencePiece | Gemini, LLaMA | 공백 없는 언어(일본어/중국어)에 강함 |

### 사전에 없는 단어 (OOV) 처리
초기 word-level 토크나이저는 모르는 단어를 `<UNK>` 로 표기하고 처리하지 못했다. 하지만 현대 토크나이저는 **서브워드(sub-word) → 글자 → 바이트** 순으로 분해할 수 있어 OOV 문제가 거의 사라졌다.

| 입력 | 분해 결과 (대략) | 비고 |
|------|------------------|------|
| `pneumonoultramicroscopicsilicovolcanoconiosis` | 약 15~20개 sub-token | 학술 용어, 사전 미등재 단어도 처리 |
| `밈무스마` (가상 신조어) | 자모 또는 UTF-8 바이트 | 한글 신조어 |
| `🦄` (이모지) | 2~3 바이트 토큰 | 모든 유니코드 처리 가능 |

> **주의**: 토큰화 가능 ≠ 의미 이해 가능. 학습 데이터에 없던 신조어·고유명사는 토큰은 받아들이지만 의미는 문맥으로만 추론한다. 또한 잘게 쪼개진 단어는 토큰 수가 늘어나 비용도 증가한다.

### 사전은 누가 만드나?
사전은 **LLM 회사가 모델 학습 전에 별도로 만들어서 모델에 박아넣는다**. 즉, 토크나이저 학습이 먼저, 모델 학습이 나중이며 한 번 정해진 사전은 모델 수명 동안 바뀌지 않는다.

```
거대 텍스트 코퍼스 수집
  → BPE/SentencePiece 알고리즘으로 빈도 분석 & 패턴 병합
  → 목표 사전 크기(예: 100K)까지 반복
  → 완성된 사전(vocab.json + merges.txt)을 토크나이저에 탑재
  → 모델 학습 시작 (이때부터 사전 고정)
```

회사/모델별로 사전이 다르므로 **같은 문장이어도 모델마다 토큰 수가 다르다**.

| 모델 | 토크나이저 | 사전 공개 여부 |
|------|------------|----------------|
| GPT-3.5 / GPT-4 | tiktoken (`cl100k_base`) | 공개 (`tiktoken` 라이브러리) |
| GPT-4o | tiktoken (`o200k_base`) | 공개 |
| Claude | 자체 BPE 변형 | 비공개 (`count_tokens` API로만 확인) |
| LLaMA | SentencePiece | 완전 공개 (오픈소스) |
| Gemini | SentencePiece | 부분 공개 |

> 사전이 공개된 모델은 API 호출 없이도 클라이언트에서 토큰 수를 미리 계산할 수 있다. Claude처럼 비공개인 경우 `count_tokens` API를 호출해야 정확한 수치를 얻을 수 있다.

## 토큰 분할 예시

| 텍스트 | 토큰 수 | 분할 예시 |
|--------|---------|-----------|
| `Hello` | 1 | `Hello` |
| `Hello, world!` | 4 | `Hello` / `,` / ` world` / `!` |
| `chatGPT` | 2 | `chat` / `GPT` |
| `TypeScript` | 2 | `Type` / `Script` |
| `안녕하세요` | 3~5 | 한글은 UTF-8 바이트로 쪼개져 더 많이 소비 |

대략적 환산:
- **영어**: 1토큰 ≈ 4글자 ≈ 0.75단어
- **한국어**: 영어 대비 약 2~3배 더 소비 (같은 의미여도 토큰 수가 많음)
- 1,000토큰 ≈ 영어 약 750단어, 한국어 약 400~500자

## 입력 토큰 vs 출력 토큰

| 구분 | 내용 | 비용 |
|------|------|------|
| 입력(Input) 토큰 | 프롬프트 + 시스템 메시지 + 이전 대화 + 첨부 파일 | 저렴 |
| 출력(Output) 토큰 | 모델이 생성한 응답 | 비쌈 (입력의 4~5배) |
| 캐시(Cached) 입력 토큰 | 캐시 히트된 입력 | 매우 저렴 (입력의 약 1/10) |

> **API 과금 = (input 토큰 × 단가) + (output 토큰 × 단가)** 구조이므로, 출력이 길어질수록 비용이 빠르게 늘어남.

## 컨텍스트 윈도우 (Context Window)
모델이 한 번에 처리 가능한 **최대 토큰 수** (입력 + 출력 합산).

| 모델 | 컨텍스트 윈도우 |
|------|-----------------|
| GPT-3.5 Turbo | 16K |
| GPT-4 Turbo / GPT-4o | 128K |
| Claude 3.5 Sonnet | 200K |
| Claude 4.x | 200K (1M 옵션) |
| Gemini 1.5 Pro | 1M ~ 2M |

참고: 200K 토큰 ≈ 한국어 책 약 300~400페이지 분량.

## "토큰이 녹는다"는 왜 일어나는가?
실무에서 토큰이 빠르게 소비되는 대표적 상황:

1. **대화 누적**: 채팅에서는 매 요청마다 이전 대화 전체를 다시 보내야 함 → 길어질수록 입력 토큰 폭증
2. **긴 첨부**: PDF, 긴 문서, 이미지 OCR 결과 등을 통째로 보낼 때
3. **코드베이스 읽기**: Claude Code 같은 도구가 여러 파일을 읽으면 수만 토큰 소비
4. **에이전트 루프**: 도구 호출 결과(터미널 출력 등)가 다시 입력으로 들어가며 누적
5. **출력 폭발**: 긴 코드 생성, 장문 보고서 등

## 토큰 절약 전략

| 전략 | 설명 |
|------|------|
| 프롬프트 캐싱 | 시스템 프롬프트·고정 컨텍스트를 캐시하여 재사용 시 비용 절감 |
| 컨텍스트 정리(Compaction) | 오래된 대화 요약 후 제거 |
| 모델 라우팅 | 간단한 작업은 Haiku/Mini 등 작은 모델로 분기 |
| `max_tokens` 제한 | 출력 길이 상한 설정으로 비용 폭주 방지 |
| 첨부 요약 전달 | 원본 PDF 대신 요약문만 전달 |
| 스트리밍 응답 | 사용자 체감 속도 개선 (비용 절감은 X, UX 개선) |

## 토큰 수 확인 방법

```python
# OpenAI - tiktoken
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4o")
len(enc.encode("Hello, world!"))  # 4
```

```python
# Anthropic - count_tokens API
from anthropic import Anthropic
client = Anthropic()
result = client.messages.count_tokens(
    model="claude-opus-4-7",
    messages=[{"role": "user", "content": "Hello"}]
)
print(result.input_tokens)
```

```bash
# 빠른 추정 (영어 기준)
echo -n "your text" | wc -c   # 글자수 / 4 ≈ 토큰수
```
