# Spring Boot 인코딩 설정

> 최종 업데이트: 2026-03-24 | 기준: Spring Boot 3.4, JDK 21

## 개요

백엔드 서버의 인코딩 설정은 세 계층으로 나뉜다.

```
JVM 레벨          →  파일 I/O, 시스템 스트림
Spring 레벨       →  HTTP 요청/응답
DB 레벨           →  데이터베이스 연결
```

| 구분 | 대상 | 설정 위치 |
|------|------|-----------|
| JVM | 파일 I/O, `System.out`, `new String(byte[])` | `-Dfile.encoding` |
| Spring | HTTP 요청/응답 | `server.servlet.encoding.*` |
| DB | JDBC 연결 | datasource URL 파라미터 |

## JVM 레벨

`file.encoding` 시스템 프로퍼티로 JVM 전체의 기본 인코딩을 결정한다.

```bash
java -Dfile.encoding=UTF-8 -jar app.jar
```

- JVM 시작 시에만 초기화되므로 코드 중간에 변경해도 반영되지 않는다
- **JDK 18+부터 기본값이 UTF-8**로 통일되어 별도 설정할 일이 거의 없다

## Spring 레벨

HTTP 요청/응답의 인코딩을 담당한다. 실무에서 한글 깨짐이 발생하는 대부분의 경우 이 설정으로 해결한다.

### application.yml

```yaml
server:
  servlet:
    encoding:
      charset: UTF-8
      enabled: true
      force: true       # request/response 모두 강제 적용
```

| 속성 | 기본값 | 설명 |
|------|--------|------|
| `enabled` | `true` | `CharacterEncodingFilter` 등록 여부 |
| `charset` | `UTF-8` | 사용할 문자셋 |
| `force` | `false` | request + response 모두 강제 적용 |
| `force-request` | `false` | 요청만 강제 적용 |
| `force-response` | `false` | 응답만 강제 적용 |

### 동작 흐름

```
Client 요청
    ↓
CharacterEncodingFilter (Spring Boot 자동 등록)
    ├── force=true  → charset 무조건 UTF-8 적용
    └── force=false → 클라이언트가 보낸 Content-Type의 charset 우선 사용
    ↓
Controller
    ↓
응답 Content-Type: application/json;charset=UTF-8
```

- `force: true` 설정 시 내부적으로 `CharacterEncodingFilter`가 request/response 모두에 인코딩을 강제한다
- Spring Boot 2.x+에서는 기본으로 UTF-8이 적용되어 있으나, 일부 라이브러리(springdoc 등)에서 charset이 누락되는 경우 `force: true`가 필요하다

## DB 연결 인코딩

JDBC URL에 charset을 명시하지 않으면 DB 서버 기본 설정을 따른다.

### MySQL

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb?characterEncoding=UTF-8&useUnicode=true
```

### PostgreSQL

```yaml
spring:
  datasource:
    # PostgreSQL은 DB 생성 시 인코딩이 결정되며, JDBC URL에서 별도 지정하지 않는다
    url: jdbc:postgresql://localhost:5432/mydb
```

## 트러블슈팅

### JSON 응답 한글 깨짐

`Content-Type`에 charset이 누락된 경우 발생한다.

```yaml
# 방법 1: 서블릿 인코딩 강제
server:
  servlet:
    encoding:
      force: true

# 방법 2: Spring MVC 미디어 타입 지정
spring:
  mvc:
    media-types:
      json: application/json;charset=UTF-8
```

### multipart 요청 한글 깨짐

파일 업로드 시 파일명이 깨지는 경우:

```yaml
spring:
  servlet:
    multipart:
      enabled: true
      file-size-threshold: 0
```

```java
@Bean
public FilterRegistrationBean<CharacterEncodingFilter> encodingFilter() {
    CharacterEncodingFilter filter = new CharacterEncodingFilter();
    filter.setEncoding("UTF-8");
    filter.setForceEncoding(true);

    FilterRegistrationBean<CharacterEncodingFilter> bean = new FilterRegistrationBean<>(filter);
    bean.setOrder(Ordered.HIGHEST_PRECEDENCE);  // 가장 먼저 실행
    return bean;
}
```

### 로그 한글 깨짐

Logback 설정에서 인코딩을 지정한다.

```xml
<!-- logback-spring.xml -->
<encoder>
    <charset>UTF-8</charset>
    <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger - %msg%n</pattern>
</encoder>
```
