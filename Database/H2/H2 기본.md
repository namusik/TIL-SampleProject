# H2 Database 기본

> 최종 업데이트: 2026-04-22 | 기준: H2 2.x

## 개념

**H2**는 **Java로 작성된 경량 관계형 데이터베이스**다. JVM 위에서 동작하며, JAR 파일 하나로 배포되기 때문에 **별도 설치 없이** Java 애플리케이션 안에 **임베디드(embedded)** 로 실행할 수 있는 것이 특징이다.

> 비유하자면 "휴대용 미니 냉장고" 같은 DB. 큰 냉장고(MySQL/Oracle)를 따로 설치할 필요 없이, 앱과 함께 들고 다니며 쓴다. 속도는 빠르고 간편하지만, 대규모 운영용은 아님.

## 배경/역사

- **개발자**: Thomas Mueller — HypersonicSQL(Hsqldb)의 원저자가 재설계
- **최초 릴리스**: 2005년
- **이름의 유래**: "Hypersonic 2" → "H2"
- **라이선스**: MPL 2.0 / EPL 1.0 (상용 포함 자유 사용)
- **주 용도 변화**: 초기엔 데스크톱/모바일 앱 임베디드 DB → 현재는 **Java 테스트·프로토타입·개발 환경**의 사실상 표준

## 왜 쓰는가

| 상황 | 이유 |
|------|------|
| **Spring Boot 로컬 개발** | 의존성만 추가하면 바로 동작, 설치 불필요 |
| **단위·통합 테스트** | 매번 빈 DB로 시작 가능, 테스트 격리 쉬움 |
| **프로토타입** | DB 설치 시간 0, 스키마 변경 자유 |
| **인메모리 캐시/임시 저장** | JVM 메모리에서 동작해 극도로 빠름 |
| **데모·학습 환경** | JDBC URL 하나로 끝 |

> 대규모 프로덕션 OLTP에는 쓰지 않는다. MySQL/PostgreSQL 등으로 대체.

## 실행 모드

H2의 가장 큰 특징. 같은 엔진이 **3가지 모드**로 동작한다.

| 모드 | 특징 | 데이터 지속성 | 쓰임새 |
|------|------|-------------|-------|
| **Embedded** | 앱의 JVM 안에서 실행 | 파일에 저장 가능 | 개발·테스트 |
| **In-Memory** | JVM 메모리에만 존재 | **앱 종료 시 소멸** | 테스트·휘발성 캐시 |
| **Server** | 별도 프로세스로 실행, TCP 접속 | 파일 저장 | 여러 앱이 공유 |

### 구조 비교

```
[Embedded]                 [Server]
 ┌────────────────┐         ┌─────────┐     TCP     ┌────────┐
 │  Java App      │         │ App A   │ ──────────► │ H2     │
 │  ┌──────────┐  │         └─────────┘             │ Server │
 │  │ H2 (JAR) │  │         ┌─────────┐             │ Process│
 │  └──────────┘  │         │ App B   │ ──────────► │        │
 └────────────────┘         └─────────┘             └────────┘
 같은 JVM 안                여러 앱이 공유 접속
```

## JDBC URL 형식

모드를 결정하는 것이 JDBC URL.

| 모드 | URL 예시 |
|------|---------|
| **In-Memory** | `jdbc:h2:mem:testdb` |
| **In-Memory (공유)** | `jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1` |
| **Embedded (파일)** | `jdbc:h2:~/mydb` (홈 디렉터리에 `mydb.mv.db` 파일 생성) |
| **Embedded (절대경로)** | `jdbc:h2:file:/data/mydb` |
| **Server (TCP)** | `jdbc:h2:tcp://localhost/~/mydb` |
| **Server (SSL)** | `jdbc:h2:ssl://localhost/~/mydb` |

### 자주 쓰는 파라미터

| 파라미터 | 설명 |
|---------|------|
| `DB_CLOSE_DELAY=-1` | 마지막 커넥션 닫혀도 DB 유지 (인메모리 모드에서 유용) |
| `MODE=MySQL` / `MODE=PostgreSQL` / `MODE=Oracle` | 해당 DB **호환 모드** |
| `DATABASE_TO_UPPER=FALSE` | 테이블/컬럼명 대소문자 구분 |
| `AUTO_SERVER=TRUE` | 자동으로 서버 모드 전환 (여러 프로세스 접근) |
| `INIT=RUNSCRIPT FROM 'init.sql'` | 연결 시 SQL 스크립트 실행 |

## Spring Boot에서 쓰기

### 1. 의존성 추가 (Gradle)

```groovy
dependencies {
    runtimeOnly 'com.h2database:h2'
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
}
```

### 2. `application.properties` 설정

```properties
# 인메모리 (기본값 — 설정 없어도 동작)
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.username=sa
spring.datasource.password=

# 파일 저장 (개발용)
spring.datasource.url=jdbc:h2:file:./data/mydb

# 서버 모드 (외부 H2 서버에 접속)
spring.datasource.url=jdbc:h2:tcp://localhost/~/mydb
spring.datasource.username=sa
spring.datasource.password=

# H2 콘솔 활성화
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console
```

> `http://localhost:8080/h2-console` 접속 시 내장 웹 콘솔에서 쿼리 실행 가능.

## 독립 서버로 실행

별도 H2 서버를 띄워 여러 앱이 공유하거나 원격 접속할 때.

### 설치 및 실행

https://www.h2database.com/html/download-archive.html 에서 스프링 의존성 버전과 맞춰 다운로드.

```sh
# 실행 권한
chmod 755 h2.sh

# 서버 시작
./h2.sh
```

- 실행 시 브라우저에 **웹 콘솔** 자동 오픈
- 설정: `Generic H2 (Server)` 선택

### 최초 DB 생성

- JDBC URL: `jdbc:h2:~/원하는_이름`
- 사용자명: `sa` (기본값) 또는 원하는 이름
- 비밀번호: 비워도 됨

DB 파일은 **사용자 홈 디렉터리(`~`)** 에 `이름.mv.db`로 생성된다. H2 설치 폴더가 아니니 주의.

### 스프링에서 접속

```properties
spring.datasource.url=jdbc:h2:tcp://localhost/~/mydb
spring.datasource.username=sa
spring.datasource.password=
```

> `./h2.sh`를 종료하면 웹 콘솔과 서버 모두 종료. 접속 중인 스프링 앱도 DB를 잃음.

자세한 설치 과정은 [H2 시작하는 법.md](H2%20시작하는%20법.md) 참고.

## 주요 기능

| 기능 | 지원 |
|------|------|
| 표준 SQL | ANSI SQL / JDBC 4.x |
| 트랜잭션 | ACID 지원 |
| 격리 수준 | READ UNCOMMITTED/READ COMMITTED/REPEATABLE READ/SERIALIZABLE |
| 암호화 | 파일 AES 암호화 |
| 풀텍스트 검색 | Lucene 기반 내장 |
| Clustering | 이중화 클러스터(제한적) |
| 트리거 / 저장 프로시저 | Java 코드로 작성 |
| **호환 모드** | MySQL, PostgreSQL, Oracle, MSSQL, DB2 등 |

### 호환 모드 예시

```
jdbc:h2:mem:test;MODE=MySQL;DATABASE_TO_LOWER=TRUE
jdbc:h2:mem:test;MODE=PostgreSQL
```

> 운영 DB가 MySQL인데 테스트는 H2로 돌릴 때 쓴다. 단, **완전 호환은 아니므로** 중요 쿼리는 실제 DB에서도 검증 필요.

## 다른 임베디드 DB와 비교

| 항목 | **H2** | **HSQLDB** | **Derby** | **SQLite** |
|------|--------|-----------|-----------|-----------|
| 언어 | Java | Java | Java | **C** |
| JVM 내 실행 | O | O | O | X (JNI) |
| 속도 | 빠름 | 빠름 | 보통 | 매우 빠름 |
| 표준 SQL 지원 | 강 | 강 | 강 | 약함(독자 확장) |
| 호환 모드 | 다양함 | 적음 | 없음 | 없음 |
| 현재 활용도 | **Spring 테스트 표준** | 일부 | Apache (저조) | 모바일·임베디드 |

## 백엔드 개발자 관점 실무 포인트

- **프로덕션 DB와 테스트 DB를 일치시키기** — H2를 테스트에 쓰면 편하지만 MySQL 전용 문법(예: `GROUP_CONCAT`) 사용 시 차이로 오류. 최근엔 **Testcontainers**로 실제 MySQL/PostgreSQL을 도커로 띄우는 방식 권장
- **스키마 초기화** — `data.sql` / `schema.sql`로 초기 데이터 주입, 또는 Flyway/Liquibase와 함께 사용
- **H2 콘솔 보안** — 운영 환경에서는 반드시 비활성화 (`spring.h2.console.enabled=false`)
- **인메모리 격리** — 테스트마다 다른 DB명 지정(`mem:test-${UUID}`)으로 병렬 테스트 충돌 방지
- **JPA ddl-auto** — 로컬 개발은 `create-drop`, 운영 연동 테스트는 `validate` 권장

## 문제 해결

| 증상 | 원인 / 해결 |
|------|-----------|
| `Database may be already in use` | 같은 파일을 다른 프로세스가 잠금 → `AUTO_SERVER=TRUE` 파라미터 |
| `Table not found` (대소문자) | H2는 기본 대문자로 저장 → `DATABASE_TO_UPPER=FALSE` 또는 호환 모드 사용 |
| 데이터가 사라짐 (인메모리) | 정상 — 앱 종료 시 소멸. 파일 모드로 전환 또는 `DB_CLOSE_DELAY=-1` |
| MySQL 쿼리가 안 먹음 | `MODE=MySQL` 추가. 그래도 안 되면 Testcontainers로 실제 MySQL 사용 |
| H2 콘솔 접속 안 됨 | `spring.h2.console.enabled=true`, 경로 확인, 보안 설정(Spring Security) 허용 |
