# 데이터베이스 아키텍처 — 클라이언트/서버 vs 임베디드

> 최종 업데이트: 2026-04-17

## 개념

데이터베이스는 애플리케이션과 어떻게 연결되느냐에 따라 크게 **클라이언트/서버 방식**과 **임베디드 방식**으로 나뉜다.

> 클라이언트/서버는 "식당에서 주문하는 것"이고, 임베디드는 "집에서 냉장고를 직접 여는 것"이다. 식당(서버)은 여러 손님을 동시에 받을 수 있지만 항상 열어둬야 하고, 냉장고(파일)는 나 혼자 쓰지만 설치할 것도 없이 바로 쓸 수 있다.

## 클라이언트/서버 방식

```
애플리케이션 ──TCP/IP──▶ DB 서버 프로세스 (데몬) ──▶ 데이터 파일
              네트워크       별도 프로세스로 항상 실행 중
```

- **별도 DB 서버 프로세스**(데몬)가 항상 실행 중이어야 한다
- 애플리케이션은 JDBC 등의 드라이버로 서버에 **네트워크 접속**(TCP/IP)하여 쿼리를 보내고 결과를 받는다
- 서버 설치, 포트 설정, 계정 생성, 모니터링 등 **운영 관리**가 필요하다

| DB | 서버 프로세스 | 기본 포트 |
|---|---|---|
| MySQL | `mysqld` | 3306 |
| PostgreSQL | `postgres` | 5432 |
| Oracle | `oracle` | 1521 |
| SQL Server | `sqlservr` | 1433 |

### 장점

- **다중 동시 접근**: 여러 애플리케이션/사용자가 동시에 읽기/쓰기 가능
- **네트워크 접근**: 원격 서버에서도 접속 가능
- **권한 관리**: GRANT/REVOKE로 세밀한 사용자별 권한 제어
- **대규모 처리**: 고부하 트래픽, TB 이상 데이터 처리에 적합

### 단점

- 서버 설치, 설정, 모니터링 등 **운영 비용** 발생
- 네트워크 통신으로 인한 **지연(latency)** 존재
- 서버 장애 시 모든 클라이언트가 영향을 받음

## 임베디드 방식

```
애플리케이션
  └─ DB 라이브러리 (앱 안에 포함) ──▶ 데이터 파일
     함수 호출로 직접 읽기/쓰기         그냥 파일 하나
```

- **별도 프로세스가 없다**. DB가 애플리케이션 안에 라이브러리로 포함되어 동작한다
- 네트워크 통신 없이 **함수 호출**로 직접 파일을 읽고 쓴다
- 서버 설치, 포트, 계정 설정이 **불필요**하다

| DB | 특징 |
|---|---|
| SQLite | 파일 기반 임베디드 DB의 대표. 단일 `.db` 파일 |
| H2 | JVM 기반 임베디드 DB. 인메모리/파일 모드. Spring Boot 테스트에서 많이 사용 |
| Derby (JavaDB) | Apache 프로젝트. JDK에 한때 기본 포함 |
| Berkeley DB | Oracle 소유. 키-값 저장소 방식 |

### 장점

- **제로 설정**: 설치, 설정, 관리 없이 바로 사용
- **이식성**: 파일 복사만으로 DB 이동/백업 가능
- **빠른 읽기**: 네트워크 오버헤드가 없어 단순 읽기는 클라이언트/서버보다 빠를 수 있음
- **경량**: 라이브러리 크기가 매우 작음 (SQLite 약 600KB)

### 단점

- **동시 쓰기 제한**: 대부분 단일 writer만 지원
- **네트워크 접근 불가**: 원격 클라이언트가 직접 접속할 수 없음
- **사용자/권한 관리 없음**: 파일 시스템 권한에 의존

## 비교 정리

| 항목 | 클라이언트/서버 | 임베디드 |
|---|---|---|
| 서버 프로세스 | 필요 (데몬 항상 실행) | 불필요 (라이브러리) |
| 통신 방식 | TCP/IP 네트워크 | 함수 호출 (in-process) |
| 동시성 | 다중 writer 지원 | 단일 writer (대부분) |
| 운영 비용 | 설치/설정/모니터링 필요 | 제로 설정 |
| 원격 접근 | 가능 | 불가 |
| 적합한 환경 | 웹 서비스, 다중 사용자, 대규모 | 모바일, 데스크톱, IoT, 테스트 |

## 백엔드 관점에서의 선택 기준

| 상황 | 선택 |
|---|---|
| 다중 사용자가 접근하는 웹 서비스 | 클라이언트/서버 (MySQL, PostgreSQL) |
| 로컬 개발/프로토타이핑 | 임베디드 (H2, SQLite) |
| 통합 테스트에서 빠른 DB 필요 | 임베디드 인메모리 (H2 `:memory:`, SQLite `:memory:`) |
| 모바일/데스크톱 앱의 로컬 저장소 | 임베디드 (SQLite) |
| 설정 데이터, 캐시 등 단일 앱 전용 저장 | 임베디드 (SQLite) |
