# .gitignore

> 최종 업데이트: 2026-04-10

## 개념

**Git이 추적(track)하지 않을 파일과 디렉토리를 지정하는 설정 파일**이다.

- 프로젝트 루트에 `.gitignore`라는 이름으로 생성
- 여기에 명시된 파일/폴더는 `git add`나 `git status`에서 **자동으로 무시**됨
- 빌드 산출물, IDE 설정, 환경변수, 의존성 폴더 등 **커밋하면 안 되는 것들**을 관리

```
Working Directory
├── src/Main.java         ← 추적 O (커밋 대상)
├── build/                ← .gitignore에 등록 → 추적 X
├── .env                  ← .gitignore에 등록 → 추적 X
├── node_modules/         ← .gitignore에 등록 → 추적 X
└── .gitignore            ← 이 파일 자체는 커밋 대상
```

> `.gitignore` 파일 자체는 **커밋하는 것이 원칙**이다. 팀원 모두가 동일한 무시 규칙을 공유해야 하기 때문이다.

---

## 무시해야 하는 파일 유형

| 유형 | 예시 | 이유 |
|------|------|------|
| **빌드 산출물** | `build/`, `target/`, `out/`, `dist/` | 소스에서 재생성 가능 |
| **의존성 폴더** | `node_modules/`, `.gradle/`, `.m2/` | 패키지 매니저가 재설치 |
| **IDE 설정** | `.idea/`, `.vscode/`, `*.iml` | 개인 환경 설정 |
| **환경변수/시크릿** | `.env`, `*.key`, `credentials.json` | 보안 위험 |
| **OS 생성 파일** | `.DS_Store`, `Thumbs.db` | 불필요한 시스템 파일 |
| **로그/임시 파일** | `*.log`, `*.tmp`, `*.swp` | 일시적 데이터 |
| **컴파일 결과** | `*.class`, `*.o`, `*.pyc` | 소스에서 재생성 가능 |

## 패턴 문법

### 기본 규칙

| 패턴 | 의미 | 예시 |
|------|------|------|
| `파일명` | 해당 이름의 파일을 모든 경로에서 무시 | `*.log` → 모든 `.log` 파일 |
| `디렉토리/` | 해당 디렉토리 전체 무시 (슬래시로 끝남) | `build/` → build 폴더 전체 |
| `/파일명` | 루트 기준 상대경로로 무시 | `/config.json` → 루트의 config.json만 |
| `!패턴` | 무시 규칙에서 **제외** (다시 추적) | `!important.log` |
| `#` | 주석 | `# 이건 주석입니다` |

### 와일드카드

| 패턴 | 의미 | 예시 |
|------|------|------|
| `*` | 슬래시를 제외한 모든 문자 (0개 이상) | `*.class` → 모든 .class 파일 |
| `**` | 0개 이상의 디렉토리 매칭 | `**/logs` → 모든 경로의 logs 폴더 |
| `?` | 임의의 한 문자 | `file?.txt` → file1.txt, fileA.txt |
| `[abc]` | 대괄호 안의 문자 중 하나 | `*.[oa]` → .o 또는 .a 파일 |
| `[0-9]` | 범위 내 문자 중 하나 | `log[0-9].txt` → log0.txt ~ log9.txt |

### 패턴 예시

```gitignore
# 모든 .log 파일 무시
*.log

# build 디렉토리 무시
build/

# 루트의 config.json만 무시 (하위 디렉토리의 config.json은 추적)
/config.json

# .env 파일 무시하되, .env.example은 추적
.env
!.env.example

# 모든 경로의 logs 디렉토리 무시
**/logs/

# src 하위 어디든 있는 .tmp 파일 무시
src/**/*.tmp
```

## 이미 추적 중인 파일 무시하기

`.gitignore`는 **아직 추적되지 않는(untracked) 파일에만 적용**된다. 이미 커밋된 파일을 나중에 `.gitignore`에 추가해도 계속 추적된다.

### 해결 방법: 캐시에서 제거

```bash
# 특정 파일을 Git 추적에서 제거 (로컬 파일은 유지)
git rm --cached 파일명

# 특정 폴더를 Git 추적에서 제거
git rm --cached -r 폴더명/

# 예시: .env 파일을 추적에서 제거
git rm --cached .env
git commit -m "stop tracking .env"
```

| 옵션 | 동작 |
|------|------|
| `git rm 파일명` | Git 추적 제거 + **로컬 파일도 삭제** |
| `git rm --cached 파일명` | Git 추적 제거 + **로컬 파일은 유지** |

> `--cached`를 빠뜨리면 실제 파일이 삭제되므로 주의해야 한다.

### 전체 캐시 초기화

대규모로 `.gitignore` 규칙을 변경한 경우, 캐시 전체를 초기화할 수 있다.

```bash
git rm -r --cached .
git add .
git commit -m "apply .gitignore rules"
```

## .gitignore 적용 범위

`.gitignore`는 여러 위치에 둘 수 있으며, 각각 적용 범위가 다르다.

| 위치 | 적용 범위 | 공유 여부 |
|------|----------|----------|
| **프로젝트 루트 `.gitignore`** | 해당 저장소 전체 | 팀 전체 (커밋됨) |
| **하위 디렉토리 `.gitignore`** | 해당 디렉토리와 하위 | 팀 전체 (커밋됨) |
| **`.git/info/exclude`** | 해당 저장소 전체 | 본인만 (커밋 안 됨) |
| **`~/.gitignore_global`** | 모든 저장소 | 본인만 (로컬 설정) |

### 글로벌 .gitignore 설정

모든 저장소에 공통으로 적용할 무시 규칙은 글로벌로 설정한다.

```bash
# 글로벌 .gitignore 파일 지정
git config --global core.excludesFile ~/.gitignore_global
```

```gitignore
# ~/.gitignore_global
.DS_Store
Thumbs.db
*.swp
*~
.idea/
.vscode/
```

> OS 파일이나 IDE 설정처럼 **개인 환경에 의존하는 파일**은 글로벌에 등록하고, **프로젝트 특성에 의존하는 파일**(build/, node_modules/)은 프로젝트 `.gitignore`에 등록하는 것이 좋다.

## 디버깅

### 특정 파일이 무시되는 이유 확인

```bash
# 어떤 .gitignore 규칙에 의해 무시되는지 확인
git check-ignore -v 파일경로

# 예시
git check-ignore -v build/output.jar
# .gitignore:3:build/    build/output.jar
# → .gitignore 3번째 줄의 "build/" 규칙에 의해 무시됨
```

### 무시된 파일 목록 확인

```bash
# 현재 무시되고 있는 모든 파일 확인
git status --ignored
```
