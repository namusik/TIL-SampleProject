# 소프트웨어 vs 하드웨어

> 최종 업데이트: 2026-03-24

## 개요

| 구분 | 정의 | 특징 | 예시 |
|------|------|------|------|
| 하드웨어 | 컴퓨터의 물리적 구성 요소 | 눈에 보이고 만질 수 있음, 물리적 변경 필요 | CPU, RAM, SSD, GPU |
| 소프트웨어 | 하드웨어에 지시를 내리는 비물리적 구성 요소 | 디지털 프로세스로 수정/배포 가능 | OS, 앱, 드라이버 |
| 펌웨어 | 하드웨어에 내장된 소프트웨어 | 하드웨어와 소프트웨어의 브릿지 역할 | BIOS, SSD 컨트롤러 |

```
소프트웨어 (Application, OS)
        ↓ 명령
펌웨어 (하드웨어 내장 소프트웨어)
        ↓ 제어
하드웨어 (CPU, 메모리, 디스크)
```

## 프로그램

특정 작업을 수행하는 일련의 명령어들의 집합. 디스크에 저장된 정적인 상태.

- **프로그램**: 디스크에 저장된 실행 파일 (정적)
- **프로세스**: 프로그램이 메모리에 로드되어 실행 중인 상태 (동적)

## 소프트웨어

하드웨어에 지시를 내려 특정 작업을 수행하도록 하는 비물리적 구성 요소. 프로그램, 데이터, 문서 등을 포함하는 넓은 개념.

| 분류 | 설명 | 예시 |
|------|------|------|
| 시스템 소프트웨어 | 하드웨어를 관리하고 응용 소프트웨어의 실행 환경 제공 | OS (Linux, Windows), 디바이스 드라이버 |
| 응용 소프트웨어 | 사용자의 특정 작업을 수행 | 웹 브라우저, Spring Boot 애플리케이션 |

## 하드웨어

컴퓨터의 물리적 구성 요소. 수정/업그레이드하려면 물리적 변경이 필요하다.

| 구성 요소 | 역할 |
|----------|------|
| CPU | 명령어 연산/처리 |
| RAM | 실행 중인 프로그램과 데이터를 임시 저장 (휘발성) |
| SSD/HDD | 데이터 영구 저장 (비휘발성) |
| GPU | 그래픽 처리, 병렬 연산 |
| NIC | 네트워크 통신 |

## 펌웨어

하드웨어 장치에 내장된 소프트웨어. 하드웨어의 비휘발성 메모리(ROM, Flash)에 저장되어 있으며, 하드웨어 제조 시 설치된다.

- 상위 소프트웨어와 하드웨어 사이의 **브릿지 역할** 수행
- 일반 소프트웨어보다 수정이 어렵지만, 펌웨어 업데이트를 통해 변경 가능

| 예시 | 설명 |
|------|------|
| BIOS/UEFI | PC 부팅 시 하드웨어 초기화 및 OS 로드 |
| SSD 컨트롤러 펌웨어 | SSD 읽기/쓰기 최적화 |
| 네트워크 카드 펌웨어 | 네트워크 프로토콜 처리 |

## 백엔드 관점에서의 관계

```
사용자 요청
    ↓
응용 소프트웨어 (Spring Boot 애플리케이션)
    ↓
시스템 소프트웨어 (Linux OS, JVM)
    ↓
펌웨어 (디스크 컨트롤러, NIC 펌웨어)
    ↓
하드웨어 (CPU 연산, RAM 저장, SSD 읽기, NIC 전송)
```

서버 성능 튜닝 시 이 계층을 이해하면 병목 지점을 파악하기 쉽다.
- **CPU 병목** → 연산 최적화, 스레드 조정
- **메모리 병목** → JVM 힙 사이즈 조정, 캐시 도입
- **디스크 I/O 병목** → SSD 전환, 쿼리 최적화
- **네트워크 병목** → 응답 압축, CDN 도입
