# RDS 필수 파라미터 그룹 설정

> 최종 업데이트: 2026-04-20 | 기준: Aurora MySQL 3.x (MySQL 8.0 호환), RDS for MySQL 8.0

## 개념

**파라미터 그룹(Parameter Group)** 은 RDS/Aurora 인스턴스에 적용할 **DB 엔진 설정값 묶음**이다. MySQL의 `my.cnf`를 AWS가 관리 리소스로 추상화한 것이라고 보면 된다. 인스턴스에 직접 `my.cnf`를 수정하는 대신, 파라미터 그룹을 만들어 인스턴스에 연결하는 방식이다.

> 비유하자면 운영체제의 "설정 프로파일"과 같다. 여러 인스턴스에 **같은 프로파일을 붙였다 뗐다** 할 수 있고, 프로파일만 바꿔도 엔진 동작이 바뀐다.

## 파라미터 그룹 종류

| 종류 | 대상 |
|------|------|
| **DB Parameter Group** | 단일 인스턴스 (RDS for MySQL/PostgreSQL) |
| **DB Cluster Parameter Group** | Aurora 클러스터 공통 (writer + reader) |
| **Default Parameter Group** | AWS 기본값 — **수정 불가**, 새로 만들어 써야 함 |

> 실무에서는 **기본 그룹은 절대 쓰지 말고**, 환경별(dev/stg/prod)로 커스텀 그룹을 만들어 연결한다.

## 적용 유형 (Apply Type)

| 유형 | 재부팅 필요 | 설명 |
|------|------------|------|
| **Dynamic** | 불필요 | 값 변경 즉시 적용 |
| **Static** | 필요 | 인스턴스 재부팅 후 적용 |

> 콘솔에서 파라미터 수정 후 "pending-reboot" 상태로 남는다면 Static 파라미터다.

## 주요 필수 설정

### 1. 데드락 감지/기록

MySQL 기본값은 데드락이 발생해도 상세 정보를 로그에 남기지 않아, 원인 분석이 불가능하다. 반드시 켜두자.

| 파라미터 | 권장값 | 적용 유형 | 설명 |
|---------|-------|----------|------|
| `innodb_print_all_deadlocks` | `1` (ON) | **Dynamic** | 모든 데드락을 Error Log에 기록. 기본값 OFF |
| `innodb_deadlock_detect` | `ON` | Dynamic | InnoDB의 데드락 감지 기능 (기본 ON, 끄지 말 것) |

```sql
-- 적용 확인
SHOW VARIABLES LIKE 'innodb_print_all_deadlocks';
SHOW VARIABLES LIKE 'innodb_deadlock_detect';
```

- `innodb_print_all_deadlocks = ON`이면 Aurora MySQL의 **에러 로그(Error Log)** 에 모든 데드락 스택이 기록된다
- 파라미터는 Dynamic이라 **인스턴스 재부팅 불필요**, 변경 즉시 적용
- 데드락이 난 뒤에 켜봤자 이전 건은 확인 불가 — **운영 시작 전에 미리 켜둘 것**

### 2. 로깅 (느린 쿼리 / 일반 쿼리 / 에러)

| 파라미터 | 권장값 | 설명 |
|---------|-------|------|
| `slow_query_log` | `1` | 느린 쿼리 로그 활성화 |
| `long_query_time` | `1` (초) | 1초 이상 걸리는 쿼리 기록 (기본 10초는 너무 관대) |
| `log_queries_not_using_indexes` | `1` | 인덱스 안 타는 쿼리도 slow log에 기록 |
| `log_output` | `FILE` | 파일에 기록 (TABLE 사용 시 성능 저하) |
| `general_log` | `0` | 일반 쿼리 로그 — 운영에서는 OFF (디스크·성능 이슈). 디버깅 시에만 임시 ON |

CloudWatch Logs로 내보내려면 **클러스터 설정**에서 Export Logs를 켜야 한다. 파라미터 그룹만으로는 부족.

### 3. 타임존 / 문자셋

| 파라미터 | 권장값 | 설명 |
|---------|-------|------|
| `time_zone` | `Asia/Seoul` | 세션 기본 타임존 (애플리케이션 표준과 일치시킬 것) |
| `character_set_server` | `utf8mb4` | 이모지·4바이트 문자 지원 |
| `collation_server` | `utf8mb4_0900_ai_ci` (MySQL 8) / `utf8mb4_unicode_ci` (5.7) | 정렬/비교 규칙 |

> `utf8`은 실제로는 3바이트짜리 불완전한 UTF-8이다. **반드시 `utf8mb4`** 를 써야 이모지·일부 한자가 깨지지 않는다.

### 4. 트랜잭션 / 격리 수준

| 파라미터 | 권장값 | 설명 |
|---------|-------|------|
| `transaction_isolation` | `READ-COMMITTED` 권장 | 기본값 REPEATABLE-READ는 갭 락 이슈 유발 가능. Aurora/애플리케이션 특성에 맞춰 선택 |
| `innodb_lock_wait_timeout` | `5`~`10` (초) | 락 대기 시간. 기본 50초는 너무 김 — 빨리 실패 후 재시도가 유리 |
| `max_execution_time` | `30000` (ms) | 30초 넘는 SELECT 자동 종료 (MySQL 5.7+) |

### 5. 커넥션

| 파라미터 | 권장값 | 설명 |
|---------|-------|------|
| `max_connections` | 인스턴스 클래스별 기본값 사용 권장 | Aurora는 `{DBInstanceClassMemory/...}` 수식으로 자동 계산 |
| `wait_timeout` | `600` (초) | 유휴 커넥션 종료 시간. 너무 길면 커넥션 누수 누적 |
| `interactive_timeout` | `600` | 인터랙티브 세션 타임아웃 |

### 6. Binlog (복제/CDC용)

| 파라미터 | 권장값 | 설명 |
|---------|-------|------|
| `binlog_format` | `ROW` | CDC(Debezium/DMS) 사용 시 필수 |
| `binlog_row_image` | `FULL` | 변경 전/후 전체 이미지 기록 |
| `binlog_retention_hours` | `24`~`168` | Aurora 전용 — 클러스터에서 binlog 보관 시간 |

> Aurora는 기본적으로 binlog를 비활성화. CDC나 외부 복제가 필요하면 명시적으로 켤 것.

## 운영 팁

### 변경 사항 추적

```sh
# 특정 파라미터 그룹 현재 설정 확인
aws rds describe-db-parameters \
  --db-parameter-group-name my-mysql-pg \
  --query "Parameters[?ParameterValue!=null].[ParameterName,ParameterValue,ApplyType]" \
  --output table

# 수정 이력
aws rds describe-db-parameters \
  --db-parameter-group-name my-mysql-pg \
  --source user
```

### 재부팅 대기 파라미터 확인

```sh
aws rds describe-db-instances \
  --db-instance-identifier my-db \
  --query "DBInstances[].PendingModifiedValues"
```

### 환경별 그룹 전략

- `mysql8-prod-pg` / `mysql8-stg-pg` / `mysql8-dev-pg` 분리
- 동일 설정이 필요하면 **복제**(`copy-db-parameter-group`)해서 한쪽만 수정
- Terraform/CloudFormation으로 **IaC 관리** 권장 (콘솔에서 손으로 바꾸면 환경 간 drift 발생)

## 체크리스트 (운영 전 최소 점검)

- [ ] `innodb_print_all_deadlocks = 1`
- [ ] `slow_query_log = 1`, `long_query_time = 1`
- [ ] `character_set_server = utf8mb4`, `collation_server = utf8mb4_0900_ai_ci`
- [ ] `time_zone = Asia/Seoul`
- [ ] `innodb_lock_wait_timeout` 단축 (5~10초)
- [ ] CloudWatch Logs Export 활성화 (클러스터 설정)
- [ ] 기본 파라미터 그룹이 아닌 **커스텀 그룹** 사용
