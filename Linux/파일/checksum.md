# 체크섬 — sha512sum / sha256sum / md5sum

> 최종 업데이트: 2026-05-21 | 기준: GNU coreutils

## 개념

**체크섬(checksum)** 은 파일이나 데이터를 고정 길이의 짧은 "지문"으로 변환한 값이다. `sha512sum` 등의 명령은 **암호학적 해시 함수**로 이 지문을 계산해, 파일이 **손상되거나 변조되지 않았는지 검증**하는 데 쓴다. `sha512sum`은 SHA-2 계열로 임의 길이 입력을 **512비트(16진수 128자리)** 로 변환한다.

> 비유하자면 파일의 "지문". 내용이 단 1바이트만 바뀌어도 지문이 완전히 달라져(눈사태 효과, avalanche effect), 다운로드가 깨졌거나 누가 손댄 걸 바로 알 수 있다.

## 배경/역사

- **MD5**(Rivest, 1991), **SHA-1**(NSA/NIST, 1995)이 오래 쓰였으나, **충돌(서로 다른 입력이 같은 해시)** 공격이 실증되어 보안 용도로는 **폐기**되었다(MD5는 2004년경, SHA-1은 2017년 Google "SHAttered"로 실제 충돌 생성).
- 그 대안이 **SHA-2 계열**(NIST, 2001) — `SHA-256` / `SHA-512` 등. 현재까지 안전하며 사실상 표준이다.
- 더 최신 표준으로 **SHA-3**(2015, 구조가 다른 Keccak 기반)이 있으나, 배포 검증 현장에선 여전히 SHA-256/512가 주력이다.
- 이 명령들은 **GNU coreutils**에 포함되어 거의 모든 리눅스에 기본 존재한다.

## 명령어 계열

| 명령 | 알고리즘 | 길이 | 비고 |
|---|---|---|---|
| `md5sum` | MD5 | 128bit | **충돌 취약 — 보안용 금지**, 단순 전송 확인용만 |
| `sha1sum` | SHA-1 | 160bit | 취약, 신규 사용 권장 안 함 |
| `sha256sum` | SHA-256 | 256bit | 가장 널리 쓰이는 표준 |
| `sha512sum` | SHA-512 | 512bit | 더 길고, **64비트 CPU에선 종종 더 빠름** |

> SHA-512가 SHA-256보다 항상 느린 게 아니다. 64비트 워드 연산 기반이라 64비트 머신에서 오히려 빠른 경우가 많다.

## 기본 사용

```bash
sha512sum file.iso              # 파일의 SHA-512 해시 출력
echo -n "hello" | sha512sum     # 표준입력 해시 (-n: 끝 개행 제외)
sha512sum a.txt b.txt           # 여러 파일 한번에
```

출력은 `<해시값>␣␣<파일명>` 형식(공백 2칸):

```
9b71d224bd62...673ca7  file.iso
```

## 무결성 검증 (핵심 용도)

배포처가 함께 제공한 해시와 다운로드한 파일을 대조한다.

```bash
# 해시 파일 생성
sha512sum file.iso > file.iso.sha512

# 해시 파일로 검증 → OK / FAILED
sha512sum -c file.iso.sha512
# file.iso: OK

# 배포처가 준 단일 해시와 즉석 비교
echo "9b71d224...673ca7  file.iso" | sha512sum -c -
```

| 옵션 | 의미 |
|---|---|
| `-c` | 해시 파일과 대조해 **검증**(OK/FAILED) |
| `-b` | 바이너리 모드로 읽기 |
| `--quiet` | `-c` 시 OK는 생략하고 실패만 출력 |
| `--ignore-missing` | `-c` 시 목록에 있으나 없는 파일은 무시 |

## 흔히 헷갈리는 점

- **해시 ≠ 암호화**: 단방향이라 되돌릴 수 없고 키도 없다. "복호화"는 불가능하다.
- **무결성 확인용이지 출처 인증용은 아니다**: 공격자가 파일과 해시를 **둘 다** 바꿔치기하면 무력하다. 출처 신뢰가 필요하면 **GPG 서명**(서명자 검증)이나 **HMAC**(공유키 기반)를 함께 쓴다.
- **비밀번호 저장엔 부적합**: 너무 빨라 무차별 대입에 약하다. 비밀번호는 의도적으로 느린 **bcrypt/scrypt/Argon2** 전용 함수를 쓴다.
- **검증의 신뢰는 해시의 출처에 달렸다**: 같은 사이트에서 파일과 해시를 같이 받으면, 그 사이트가 뚫렸을 때 둘 다 위조된다. 해시는 가능하면 **다른 경로(공식 키서버·HTTPS 별도 채널)** 로 확인한다.
