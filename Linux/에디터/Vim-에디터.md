# Vim 에디터

> 최종 업데이트: 2026-04-09 | Vim 9.x / Neovim 0.10+ 기준

---

## 개념

Vim(Vi IMproved)은 Unix 계열 운영체제에서 가장 널리 사용되는 텍스트 에디터다. 1991년 네덜란드 개발자 **Bram Moolenaar**가 Amiga 컴퓨터에서 Vi를 사용하기 위해 만들었으며, 처음에는 "Vi IMitation"이라는 이름이었다가 1992년 v1.22부터 기능이 크게 확장되면서 "Vi IMproved"로 이름이 바뀌었다.

> 비유: 일반 텍스트 에디터가 "항상 글을 쓰는 상태의 펜"이라면, Vim은 "명령 모드와 쓰기 모드를 전환할 수 있는 만능 도구"다. 처음에는 불편하지만, 손에 익으면 마우스 없이 키보드만으로 모든 편집을 할 수 있다.

### Vi와의 관계

| 구분 | Vi | Vim |
|------|-----|-----|
| 개발자 | Bill Joy (1976) | Bram Moolenaar (1991) |
| 목적 | Unix의 기본 텍스트 에디터 | Vi의 기능 확장 버전 |
| 시각적 피드백 | 제한적 | 구문 강조, 다중 실행 취소 등 지원 |
| 플러그인 | 없음 | 풍부한 플러그인 생태계 |
| 현재 상태 | 대부분의 시스템에서 Vim이 vi로 심볼릭 링크됨 | 사실상 표준 |

## 핵심 철학 — 모달 에디터 (Modal Editor)

Vim의 가장 큰 특징은 **모드(mode) 기반 편집**이다. 대부분의 에디터는 키를 누르면 바로 글자가 입력되지만, Vim은 현재 어떤 모드에 있느냐에 따라 같은 키가 완전히 다른 동작을 수행한다.

> 비유: TV 리모컨에 "음량 조절 모드"와 "채널 변경 모드"가 있다고 생각하면 된다. 같은 숫자 버튼이지만 모드에 따라 음량을 바꾸거나 채널을 바꾸는 것처럼, Vim에서 `d`는 Normal 모드에서는 삭제, Insert 모드에서는 그냥 문자 `d`를 입력한다.

### 왜 모달인가?

프로그래머의 작업 시간 대부분은 **새 텍스트를 작성하는 것이 아니라 읽고, 탐색하고, 수정하는 것**이다. Vim은 이 관찰에 기반해 "탐색/조작"이 기본 모드(Normal)이고, "입력"은 별도 모드(Insert)로 분리했다. 이 설계 덕분에 마우스 없이 키보드만으로 매우 빠르게 편집할 수 있다.

## 4가지 주요 모드 상세

### 1. Normal 모드 (기본 모드)

Vim을 실행하면 처음 진입하는 모드. 텍스트를 **탐색하고 조작**하는 데 사용한다. 커서 이동, 복사, 삭제, 붙여넣기 등 대부분의 명령이 이 모드에서 동작한다.

- 다른 모드에서 `Esc`를 누르면 항상 Normal 모드로 돌아온다
- 가장 많이 사용하는 모드이므로 기본 모드로 설정되어 있다

### 2. Insert 모드 (입력 모드)

일반 텍스트 에디터처럼 **키를 누르면 글자가 입력**되는 모드. 메모장이나 VS Code의 기본 동작과 동일하다.

| 진입 키 | 동작 |
|---------|------|
| `i` | 커서 **앞**에서 입력 시작 |
| `a` | 커서 **뒤**에서 입력 시작 |
| `o` | 현재 줄 **아래**에 새 줄 삽입 후 입력 |
| `I` | 줄 **맨 앞**에서 입력 시작 |
| `A` | 줄 **맨 뒤**에서 입력 시작 |
| `O` | 현재 줄 **위**에 새 줄 삽입 후 입력 |

### 3. Visual 모드 (선택 모드)

텍스트를 **시각적으로 선택**(드래그)하여 복사, 삭제, 치환 등의 작업을 하는 모드. 마우스로 텍스트를 드래그하는 것과 같은 개념이다.

| 진입 키 | 선택 방식 |
|---------|----------|
| `v` | **문자 단위** 선택 — 한 글자씩 범위를 늘려간다 |
| `V` | **줄 단위** 선택 — 줄 전체를 한 번에 선택 |
| `Ctrl+v` | **블록(사각형) 단위** 선택 — 여러 줄의 특정 열만 선택 |

### 4. Command-Line 모드 (명령줄 모드)

화면 하단에 명령어를 직접 입력하여 실행하는 모드. 저장, 종료, 검색/치환, 설정 변경 등 고급 기능을 수행한다.

- Normal 모드에서 `:`을 누르면 진입
- `/`(앞으로 검색)이나 `?`(뒤로 검색)으로도 진입
- 명령 실행 후 자동으로 Normal 모드로 복귀

## 모드 간 전환 다이어그램

```
                          i, a, o, I, A, O
               ┌─────────────────────────────────┐
               │                                 ▼
         ┌───────────┐                    ┌───────────┐
         │           │◀──── Esc ──────────│           │
         │  Normal   │                    │  Insert   │
         │  (기본)    │                    │  (입력)    │
         │           │────── Esc ────────▶│           │
         └───────────┘                    └───────────┘
           │       ▲                           
           │       │ Esc                       
           │       │                           
  v, V,    │       │         : / ? 
  Ctrl+v   │    ┌──┴──────────┐    ┌─────────────────┐
           │    │             │    │                 │
           ▼    │             │◀───│  Command-Line   │
         ┌───────────┐       │    │  (명령줄)        │
         │           │───────┘    └─────────────────┘
         │  Visual   │                  ▲
         │  (선택)    │                  │
         │           │    Normal에서 : 또는 / 또는 ?
         └───────────┘
```

**핵심 원칙**: 어떤 모드에서든 `Esc`를 누르면 Normal 모드로 돌아간다.

## 커서 이동

> Vim의 이동 명령은 hjkl을 기본으로 하되, 숫자와 조합하면 배수로 이동한다. 예: `5j` = 아래로 5줄 이동

### 기본 이동 (hjkl)

| 키 | 방향 | 외우는 법 |
|----|------|----------|
| `h` | ← 왼쪽 | 키보드에서 가장 왼쪽 |
| `j` | ↓ 아래 | `j`의 꼬리가 아래로 향함 |
| `k` | ↑ 위 | `k`의 획이 위로 올라감 |
| `l` | → 오른쪽 | 키보드에서 가장 오른쪽 |

### 단어 단위 이동

| 키 | 동작 |
|----|------|
| `w` | 다음 단어의 **시작**으로 (word) |
| `b` | 이전 단어의 **시작**으로 (back) |
| `e` | 현재/다음 단어의 **끝**으로 (end) |
| `W` | 다음 WORD의 시작 (공백 기준, 특수문자 포함) |
| `B` | 이전 WORD의 시작 |
| `E` | 현재/다음 WORD의 끝 |

> `w`는 단어를 구두점 기준으로 나누고, `W`는 공백 기준으로만 나눈다. 예: `hello-world`에서 `w`는 `hello`, `-`, `world`를 각각 단어로 보지만, `W`는 전체를 하나의 WORD로 본다.

### 줄 내 이동

| 키 | 동작 |
|----|------|
| `0` | 줄의 맨 처음 (0번째 칼럼) |
| `^` | 줄의 첫 번째 비공백 문자 |
| `$` | 줄의 맨 끝 |
| `f{문자}` | 현재 줄에서 해당 문자를 앞으로 찾아 이동 |
| `F{문자}` | 현재 줄에서 해당 문자를 뒤로 찾아 이동 |
| `t{문자}` | 해당 문자 바로 앞까지 이동 |
| `T{문자}` | 해당 문자 바로 뒤까지 이동 (역방향) |
| `;` | `f/t` 검색 반복 (같은 방향) |
| `,` | `f/t` 검색 반복 (반대 방향) |

### 파일 내 이동

| 키 | 동작 |
|----|------|
| `gg` | 파일 맨 처음으로 |
| `G` | 파일 맨 끝으로 |
| `{숫자}G` | 해당 줄 번호로 이동 (예: `42G` → 42번째 줄) |
| `:{숫자}` | 해당 줄 번호로 이동 (예: `:42`) |
| `H` | 화면 상단(High)으로 |
| `M` | 화면 중앙(Middle)으로 |
| `L` | 화면 하단(Low)으로 |

### 스크롤

| 키 | 동작 |
|----|------|
| `Ctrl+f` | 한 화면 아래로 (forward) |
| `Ctrl+b` | 한 화면 위로 (backward) |
| `Ctrl+d` | 반 화면 아래로 (down) |
| `Ctrl+u` | 반 화면 위로 (up) |
| `zz` | 현재 줄을 화면 중앙으로 |
| `zt` | 현재 줄을 화면 상단으로 (top) |
| `zb` | 현재 줄을 화면 하단으로 (bottom) |

## 입력/편집

> Normal 모드에서 아래 키를 누르면 Insert 모드로 진입하거나 텍스트를 직접 수정한다

### Insert 모드 진입

| 키 | 동작 |
|----|------|
| `i` | 커서 **앞**에서 입력 (insert) |
| `a` | 커서 **뒤**에서 입력 (append) |
| `o` | 아래에 새 줄 추가 후 입력 (open below) |
| `I` | 줄 맨 앞에서 입력 |
| `A` | 줄 맨 뒤에서 입력 |
| `O` | 위에 새 줄 추가 후 입력 (open above) |

### 직접 수정 (Normal 모드 유지 또는 즉시 Insert 진입)

| 키 | 동작 |
|----|------|
| `r{문자}` | 커서 위의 한 문자를 교체 (Normal 모드 유지) |
| `R` | Replace 모드 진입 — 타이핑하면 기존 문자를 덮어씀 |
| `s` | 커서 위의 한 문자를 삭제하고 Insert 모드 진입 (substitute) |
| `S` | 현재 줄 전체를 삭제하고 Insert 모드 진입 |
| `c{동작}` | 동작 범위만큼 삭제하고 Insert 모드 진입 (change) |
| `cw` | 단어 끝까지 삭제 후 입력 |
| `c$` 또는 `C` | 줄 끝까지 삭제 후 입력 |
| `cc` | 줄 전체 삭제 후 입력 (`S`와 동일) |
| `ci"` | 따옴표 안의 내용을 삭제 후 입력 (change inside ") |
| `ci(` | 괄호 안의 내용을 삭제 후 입력 |

> `c`(change)는 "삭제 + Insert 모드 진입"이 합쳐진 명령이다. `d`(delete)와 동일한 범위 지정을 사용하되, 삭제 후 바로 입력할 수 있다는 차이가 있다.

## 삭제

> 삭제된 내용은 Vim의 레지스터(클립보드)에 자동 저장되므로 `p`로 붙여넣기 가능하다. 즉, Vim에서 삭제는 "잘라내기(cut)"와 같다.

| 키 | 동작 |
|----|------|
| `x` | 커서 위의 한 문자 삭제 |
| `X` | 커서 앞의 한 문자 삭제 (백스페이스와 유사) |
| `dd` | 현재 줄 전체 삭제 |
| `dw` | 커서 위치부터 단어 끝까지 삭제 |
| `db` | 커서 위치부터 단어 시작까지 삭제 |
| `d$` 또는 `D` | 커서 위치부터 줄 끝까지 삭제 |
| `d0` | 커서 위치부터 줄 처음까지 삭제 |
| `d^` | 커서 위치부터 첫 비공백 문자까지 삭제 |
| `dgg` | 커서 위치부터 파일 처음까지 삭제 |
| `dG` | 커서 위치부터 파일 끝까지 삭제 |
| `di"` | 따옴표 안의 내용만 삭제 (delete inside ") |
| `da"` | 따옴표 포함하여 삭제 (delete around ") |
| `{숫자}dd` | N줄 삭제 (예: `5dd` → 현재 줄 포함 5줄 삭제) |

### 동사 + 명사 조합

Vim 명령어는 **동사(verb) + 명사(noun)** 구조로 조합할 수 있다.

```
d (삭제)  +  w (단어)    = dw  (단어 삭제)
d (삭제)  +  $ (줄 끝)   = d$  (줄 끝까지 삭제)
c (변경)  +  w (단어)    = cw  (단어 변경)
y (복사)  +  y (현재 줄)  = yy  (현재 줄 복사)
```

이 조합 규칙 덕분에 명령어를 외우는 것이 아니라 **문법처럼 이해**할 수 있다.

## 복사/붙여넣기

> Vim에서 복사는 **yank**이라고 부른다. `y`가 복사, `p`가 붙여넣기.

| 키 | 동작 |
|----|------|
| `yy` | 현재 줄 전체 복사 |
| `yw` | 커서 위치부터 단어 끝까지 복사 |
| `y$` | 커서 위치부터 줄 끝까지 복사 |
| `y0` | 커서 위치부터 줄 처음까지 복사 |
| `ygg` | 커서 위치부터 파일 처음까지 복사 |
| `yG` | 커서 위치부터 파일 끝까지 복사 |
| `{숫자}yy` | N줄 복사 (예: `3yy` → 3줄 복사) |
| `p` | 커서 **뒤**(아래 줄)에 붙여넣기 (paste) |
| `P` | 커서 **앞**(위 줄)에 붙여넣기 |

### Visual 모드와 조합

```
v로 선택 → y → 원하는 위치로 이동 → p
```

Visual 모드에서 범위를 선택한 뒤 `y`로 복사하거나 `d`로 삭제할 수 있다.

### 시스템 클립보드 연동

```vim
"+y     " 선택한 텍스트를 시스템 클립보드로 복사
"+p     " 시스템 클립보드에서 붙여넣기
```

> Vim은 자체 레지스터(클립보드)를 사용하므로 기본적으로 OS 클립보드와 분리되어 있다. `"+` 레지스터가 시스템 클립보드에 해당한다. `:set clipboard=unnamedplus` 설정으로 기본 연동할 수 있다.

## 되돌리기 / 다시하기

| 키 | 동작 |
|----|------|
| `u` | 실행 취소 (undo) — 마지막 변경을 되돌린다 |
| `U` | 현재 줄의 모든 변경을 되돌린다 |
| `Ctrl+r` | 다시 실행 (redo) — 되돌린 것을 다시 적용 |
| `.` | 마지막 명령 반복 (가장 유용한 키 중 하나) |

> `.`(dot) 명령은 Vim의 숨겨진 보석이다. 예를 들어 `cw`로 단어를 변경한 뒤, 다른 단어에서 `.`을 누르면 같은 변경이 반복된다. 반복 작업을 극적으로 줄여준다.

Vim은 **실행 취소 트리(undo tree)**를 지원한다. 일반적인 에디터는 실행 취소가 직선형(선형)이지만, Vim은 분기(branch)를 가진 트리 구조이므로 어떤 변경 지점으로든 돌아갈 수 있다.

## 검색/치환

### 검색

| 키 | 동작 |
|----|------|
| `/패턴` | 앞으로(아래로) 검색 |
| `?패턴` | 뒤로(위로) 검색 |
| `n` | 같은 방향으로 다음 검색 결과 |
| `N` | 반대 방향으로 다음 검색 결과 |
| `*` | 커서 위의 단어를 앞으로 검색 |
| `#` | 커서 위의 단어를 뒤로 검색 |

### 치환 (찾아 바꾸기)

```vim
:s/old/new/          " 현재 줄에서 첫 번째 old → new
:s/old/new/g         " 현재 줄에서 모든 old → new (global)
:%s/old/new/g        " 파일 전체에서 모든 old → new
:%s/old/new/gc       " 파일 전체에서 하나씩 확인하며 치환 (confirm)
:10,20s/old/new/g    " 10~20번째 줄 범위에서 치환
:'<,'>s/old/new/g    " Visual 모드로 선택한 범위에서 치환
:%s/old/new/gi       " 대소문자 무시하고 치환 (case insensitive)
```

### 치환 플래그 정리

| 플래그 | 의미 |
|--------|------|
| `g` | 줄에서 모든 매칭을 치환 (없으면 첫 번째만) |
| `c` | 각 매칭마다 확인 (y/n/a/q/l) |
| `i` | 대소문자 무시 |
| `I` | 대소문자 구분 강제 |

## 저장/종료

| 명령 | 동작 |
|------|------|
| `:w` | 저장 (write) |
| `:w 파일명` | 다른 이름으로 저장 |
| `:q` | 종료 (변경사항 없을 때) |
| `:q!` | 저장하지 않고 강제 종료 |
| `:wq` | 저장 후 종료 |
| `:wq!` | 강제 저장 후 종료 (읽기 전용 파일 등) |
| `:x` | 변경사항이 있으면 저장 후 종료 (`:wq`와 유사하되 변경 없으면 저장 안 함) |
| `ZZ` | Normal 모드에서 저장 후 종료 (`:x`와 동일) |
| `ZQ` | Normal 모드에서 저장하지 않고 종료 (`:q!`와 동일) |
| `:wa` | 열린 모든 파일 저장 |
| `:qa` | 모든 창 종료 |
| `:qa!` | 모든 창 강제 종료 |

> 서버에서 실수로 Vim에 진입했을 때 당황하지 않으려면 `:q!`만 기억하면 된다. "Vim을 어떻게 종료하나요?"는 실제로 Stack Overflow에서 수백만 조회수를 기록한 질문이다.

## 변경 이력 확인

### :changes — 변경 목록

```vim
:changes
```

현재 버퍼에서 수행된 변경 목록을 보여준다. 각 변경의 줄 번호, 칼럼, 변경 내용을 확인할 수 있다.

```
change line  col text
    3    15    4 some changed text
    2    22    0 another change
    1     8   12 most recent change
>
```

| 키 | 동작 |
|----|------|
| `g;` | 이전 변경 위치로 이동 |
| `g,` | 다음 변경 위치로 이동 |

### :undolist — 실행 취소 분기 목록

```vim
:undolist
```

Vim의 undo tree에서 각 분기(leaf)의 변경 횟수와 시간을 보여준다.

```
number changes  when               saved
     3       2  2026/04/09 10:30:00
     5       4  2026/04/09 10:32:15    1
```

### 시간 기반 되돌리기

```vim
:earlier 10m     " 10분 전 상태로 되돌리기
:later 5m        " 5분 뒤 상태로 이동
:earlier 3f      " 3번의 파일 저장 전 상태로 되돌리기
```

> 이 기능은 Vim만의 강력한 특징이다. 다른 에디터에서는 되돌리기가 선형적이지만, Vim은 시간을 기준으로 원하는 과거 상태로 정확히 돌아갈 수 있다.

### 버퍼

> Vim에서 파일을 열면 메모리에 **버퍼(buffer)**가 생성된다. 창(window)은 버퍼를 보여주는 뷰포트이고, 탭(tab)은 창의 레이아웃이다.

| 명령 | 동작 |
|------|------|
| `:ls` 또는 `:buffers` | 열린 버퍼 목록 |

| `:bp` | 이전 버퍼 (buffer previous) |
| `:bd` | 현재 버퍼 삭제 (buffer delete) |

│  │            │  │            │         │

## .vimrc 기본 설정 예시

`.vimrc`는 Vim이 실행될 때 자동으로 읽는 설정 파일이다. 홈 디렉토리(`~/.vimrc`)에 위치한다.

> 비유: `.vimrc`는 Vim의 "환경 설정 파일"이다. 스마트폰의 설정 앱처럼, Vim의 모든 동작을 사용자 취향에 맞게 바꿀 수 있다.

```vim
" ===== 기본 설정 =====
set nocompatible          " Vi 호환 모드 비활성화 (Vim 기능 전체 사용)
filetype plugin indent on " 파일 타입별 플러그인 및 들여쓰기 활성화
syntax on                 " 구문 강조

" ===== 화면 표시 =====
set number                " 줄 번호 표시
set relativenumber        " 상대 줄 번호 (이동할 줄 수를 쉽게 파악)
set cursorline            " 현재 줄 강조
set showcmd               " 입력 중인 명령어 표시
set showmode              " 현재 모드 표시
set laststatus=2          " 항상 상태바 표시
set signcolumn=yes        " 좌측 사인 칼럼 항상 표시

" ===== 들여쓰기 =====
set tabstop=4             " 탭 문자를 4칸으로 표시
set shiftwidth=4          " 자동 들여쓰기 4칸
set softtabstop=4         " 편집 시 탭을 4칸으로 처리
set expandtab             " 탭을 스페이스로 변환
set autoindent            " 새 줄에서 들여쓰기 유지
set smartindent           " 코드 문맥에 따른 자동 들여쓰기

" ===== 검색 =====
set hlsearch              " 검색 결과 강조
set incsearch             " 실시간 검색
set ignorecase            " 검색 시 대소문자 무시
set smartcase             " 대문자 포함 시 대소문자 구분

" ===== 편집 =====
set encoding=utf-8        " UTF-8 인코딩
set backspace=indent,eol,start  " 백스페이스 정상 동작
set hidden                " 저장하지 않은 버퍼 허용
set mouse=a               " 마우스 사용 활성화
set clipboard=unnamedplus " 시스템 클립보드 연동
set scrolloff=8           " 커서 위아래 8줄 여유
set sidescrolloff=8       " 커서 좌우 8칸 여유

" ===== 자동완성 =====
set wildmenu              " 명령줄 자동완성 메뉴
set wildmode=longest:full,full

" ===== 성능/안정성 =====
set updatetime=300        " 스왑 파일 갱신 주기 (ms)
set timeoutlen=500        " 키 조합 대기 시간 (ms)
set undofile              " 실행 취소 이력을 파일로 영구 보존
set undodir=~/.vim/undodir

" ===== 유용한 키 매핑 =====
" 리더 키를 스페이스로 설정
let mapleader = " "

" jk를 연속 입력하면 Insert 모드 탈출
inoremap jk <Esc>

" 검색 강조 끄기
nnoremap <leader>h :noh<CR>

" 창 이동을 Ctrl+hjkl로
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l

" 줄 이동 (Alt+j/k)
nnoremap <A-j> :m .+1<CR>==
nnoremap <A-k> :m .-2<CR>==
```

## Vim vs Nano vs Emacs 비교

| 항목 | Vim | Nano | Emacs |
|------|-----|------|-------|
| 개발 시작 | 1991 | 1999 | 1976 |
| 편집 방식 | 모달 (모드 기반) | 비모달 (일반 에디터와 동일) | 비모달 (조합 키 기반) |
| 학습 곡선 | 가파름 (모드 개념 적응 필요) | 매우 낮음 (바로 사용 가능) | 가파름 (복잡한 키 조합) |
| 설치 여부 | 대부분의 Linux에 기본 설치 | 많은 배포판에 기본 설치 | 별도 설치 필요한 경우 많음 |
| 확장성 | VimScript / Vim9 script | 매우 제한적 | Emacs Lisp (거의 무한) |
| 리소스 사용 | 매우 가벼움 | 매우 가벼움 | 상대적으로 무거움 |
| 커스터마이징 | `.vimrc` + 풍부한 플러그인 | `.nanorc` (제한적) | `.emacs` / `init.el` (무한) |
| 주요 사용자 | 서버 관리자, 개발자 | 초보자, 빠른 수정 | Lisp 개발자, 올인원 환경 |
| 철학 | "한 가지를 잘 하자" | "단순하게 쓰자" | "에디터가 아니라 OS다" |

### 추천 가이드

- **서버 관리자/백엔드 개발자**: Vim을 익혀두면 어떤 서버에서도 빠르게 파일을 편집할 수 있다. 거의 모든 Linux/Unix 시스템에 설치되어 있다.
- **빠른 수정만 필요한 경우**: Nano로 충분하다. 5분이면 사용법을 익힐 수 있다.
- **올인원 개발 환경을 원하는 경우**: Emacs는 이메일, 캘린더, 파일 관리 등 거의 모든 것을 에디터 안에서 할 수 있다.

## Neovim 소개

Neovim은 2014년에 시작된 **Vim의 현대적 포크(fork)**다. Vim의 코드베이스가 오래되어 유지보수가 어려워지자, 코드를 리팩토링하고 현대적 기능을 추가하기 위해 만들어졌다.

> 비유: Vim이 "검증된 명품 자동차"라면, Neovim은 "같은 차체에 최신 전기 엔진을 넣은 버전"이다. 운전 방법(키바인딩)은 같지만 내부가 현대화되었다.

### Vim과의 주요 차이점

| 항목 | Vim | Neovim |
|------|-----|--------|
| 설정 언어 | VimScript / Vim9 script | **Lua** + VimScript 모두 지원 |
| LSP 지원 | 플러그인 필요 (coc.nvim 등) | **내장 LSP 클라이언트** |
| 비동기 처리 | Vim 8부터 제한적 지원 | **네이티브 비동기 처리** |
| 트리시터 | 미지원 | **Tree-sitter 내장** (정확한 구문 강조) |
| 터미널 | Vim 8부터 지원 | 개선된 내장 터미널 |
| 플러그인 성능 | VimScript 기반 (인터프리터) | Lua 기반 (JIT 컴파일, 더 빠름) |
| 설정 파일 | `~/.vimrc` | `~/.config/nvim/init.lua` |
| GitHub Stars | ~37,000 | ~98,000 |
| 커뮤니티 | 안정적 유지보수 | 활발한 개발 |

### Neovim의 핵심 기능

- **Lua 기반 설정/플러그인**: VimScript보다 빠르고 범용적인 Lua로 설정과 플러그인을 작성할 수 있다
- **내장 LSP (Language Server Protocol)**: 별도 플러그인 없이 코드 자동완성, 정의 이동, 리팩토링 등 IDE급 기능을 사용할 수 있다
- **Tree-sitter**: 소스 코드를 AST(추상 구문 트리)로 파싱하여 더 정확한 구문 강조와 코드 탐색을 제공한다
- **인기 배포판**: LazyVim, NvChad, AstroNvim 등 사전 설정된 배포판으로 즉시 IDE처럼 사용할 수 있다

### 설치

```bash
# macOS
brew install neovim

# Ubuntu/Debian
sudo apt install neovim

# 실행
nvim
```

### 어떤 것을 선택할까?

- **Vim**: 이미 Vim에 익숙하고 안정성을 중시하거나, 서버 환경에서 기본 설치된 에디터를 사용해야 할 때
- **Neovim**: 새로 시작하거나, IDE급 개발 환경을 터미널 에디터로 구축하고 싶을 때. Lua 기반 현대적 플러그인 생태계를 활용하고 싶을 때
