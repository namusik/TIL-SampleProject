# Linux 파일 관리 명령어

> 2026-04 기준 정리

Linux에서 파일과 디렉토리를 다루는 핵심 명령어 모음. 서버 운영 시 가장 빈번하게 사용하는 명령어들이다.

---

## 파일/디렉토리 목록 조회 (ls)

> 서랍을 열어서 안에 뭐가 있는지 보는 것

```bash
ls              # 현재 디렉토리 파일 목록
ls -l           # 상세 정보 (권한, 소유자, 크기, 수정일)
ls -a           # 숨김 파일(.으로 시작) 포함
ls -la          # 상세 + 숨김 파일
ls -lh          # 파일 크기를 사람이 읽기 쉬운 단위로 (KB, MB)
ls -lt          # 수정 시간 기준 정렬 (최신순)
ls -lS          # 파일 크기 기준 정렬 (큰 순)
ls -R           # 하위 디렉토리까지 재귀적으로 표시
```

### ls -l 출력 구조

```
-rw-r--r--  1  user  group  4096  Apr 03 10:00  file.txt
│           │  │     │      │     │              └─ 파일명
│           │  │     │      │     └─ 수정 시간
│           │  │     │      └─ 파일 크기 (bytes)
│           │  │     └─ 소유 그룹
│           │  └─ 소유자
│           └─ 하드 링크 수
└─ 파일 타입 + 권한 (d: 디렉토리, -: 일반 파일, l: 심볼릭 링크)
```

## 파일 생성 (touch)

> 빈 노트를 하나 새로 만드는 것. 이미 존재하면 수정 시간만 갱신

```bash
touch file.txt               # 빈 파일 생성 (이미 있으면 수정 시간 갱신)
touch file1.txt file2.txt    # 여러 파일 동시 생성
touch -t 202604031200 f.txt  # 특정 시간으로 타임스탬프 설정
```

## 디렉토리 생성 (mkdir)

```bash
mkdir mydir               # 디렉토리 생성
mkdir -p a/b/c            # 중간 경로까지 한 번에 생성
mkdir -m 755 mydir        # 권한을 지정하면서 생성
```

- `-p` : 부모 디렉토리가 없어도 에러 없이 전체 경로를 생성. 실무에서 거의 항상 사용

## 파일 복사 (cp)

> 원본은 그대로 두고, 사본을 만드는 것

```bash
cp file.txt /backup/                   # 파일 복사
cp file.txt /backup/new_name.txt       # 이름 변경하면서 복사
cp -r mydir/ /backup/                  # 디렉토리 전체 복사 (재귀)
cp -i file.txt /backup/               # 덮어쓰기 전 확인 (interactive)
cp -p file.txt /backup/               # 권한, 타임스탬프 보존
cp -a mydir/ /backup/                 # 아카이브 모드 (-r + -p + 심볼릭 링크 보존)
```

| 옵션 | 의미 |
|------|------|
| `-r` | 디렉토리 재귀 복사 |
| `-i` | 덮어쓰기 전 확인 |
| `-p` | 파일 속성(권한, 시간) 보존 |
| `-a` | 아카이브 모드 (`-rp` + 심볼릭 링크 보존) |
| `-v` | 복사 과정 출력 |

## 파일/디렉토리 이동 및 이름 변경 (mv)

> 파일을 다른 서랍으로 옮기거나, 이름표를 바꾸는 것. Linux에서는 "이동"과 "이름 변경"이 같은 명령어

```bash
mv old.txt new.txt                  # 이름 변경
mv file.txt /home/user/backup/     # 파일 이동
mv mydir/ /home/user/backup/       # 디렉토리 이동
mv -i file.txt /backup/            # 덮어쓰기 전 확인
mv -v old.txt new.txt              # 이동 과정 출력
```

- `cp`와 달리 원본이 사라진다
- 같은 파일 시스템 내 이동은 메타데이터만 변경되므로 매우 빠름

## 파일 삭제 (rm)

> 휴지통 없이 바로 삭제. 복구가 어려우므로 주의

```bash
rm file.txt                # 파일 삭제
rm -i file.txt             # 삭제 전 확인
rm -f file.txt             # 강제 삭제 (확인 없이)
rm -r mydir/               # 디렉토리 및 하위 내용 전체 삭제
rm -rf mydir/              # 디렉토리 강제 삭제
```

| 옵션 | 의미 |
|------|------|
| `-i` | 삭제 전 확인 |
| `-f` | 강제 삭제 (에러 무시) |
| `-r` | 디렉토리 재귀 삭제 |
| `-v` | 삭제 과정 출력 |

### 주의사항

```bash
# 절대 실행하면 안 되는 명령어
rm -rf /          # 루트 디렉토리 전체 삭제 시도 (시스템 파괴)
rm -rf /*         # 위와 동일한 효과
rm -rf ~          # 홈 디렉토리 전체 삭제
```

- 실무에서는 `rm -rf`를 사용하기 전 `ls`로 대상을 먼저 확인하는 습관이 중요
- 변수 사용 시 특히 주의: `rm -rf $DIR/` → `$DIR`이 비어있으면 `rm -rf /`가 될 수 있음

## 빈 디렉토리 삭제 (rmdir)

```bash
rmdir emptydir            # 빈 디렉토리만 삭제 (내용물이 있으면 에러)
rmdir -p a/b/c            # c → b → a 순으로 빈 디렉토리 연쇄 삭제
```

- `rm -r`과 달리 **비어있지 않으면 삭제되지 않아 안전**

## 파일 내용 보기

### cat - 전체 출력

> 짧은 파일 내용을 한 번에 보는 것

```bash
cat file.txt              # 파일 내용 전체 출력
cat -n file.txt           # 줄 번호 표시
cat file1 file2 > merged  # 여러 파일 합치기
```

### head / tail - 앞/뒤 일부만 보기

```bash
head file.txt             # 앞 10줄 (기본값)
head -n 20 file.txt       # 앞 20줄
tail file.txt             # 뒤 10줄
tail -n 20 file.txt       # 뒤 20줄
tail -f app.log           # 실시간 로그 모니터링 (파일이 추가되면 자동 출력)
```

- `tail -f`는 서버 로그 실시간 확인에 필수. 파이프로 `grep`과 조합하면 특정 패턴만 모니터링 가능

```bash
tail -f app.log | grep "ERROR"    # ERROR가 포함된 로그만 실시간 확인
```

### less - 페이지 단위로 보기

> 큰 파일을 스크롤하면서 읽는 것. `cat`은 전체를 한 번에 쏟아내지만, `less`는 한 페이지씩 보여준다

```bash
less file.txt
```

| 키 | 동작 |
|-----|------|
| `Space` / `f` | 다음 페이지 |
| `b` | 이전 페이지 |
| `/검색어` | 아래로 검색 |
| `?검색어` | 위로 검색 |
| `n` | 다음 검색 결과 |
| `N` | 이전 검색 결과 |
| `g` | 파일 처음으로 |
| `G` | 파일 끝으로 |
| `q` | 종료 |

## 파일 검색 (find)

> 파일 시스템 전체를 뒤져서 조건에 맞는 파일을 찾는 것

```bash
find /path -name "*.log"                  # 이름으로 검색
find /path -iname "*.log"                 # 대소문자 무시 검색
find /path -type f                        # 파일만 검색
find /path -type d                        # 디렉토리만 검색
find /path -size +100M                    # 100MB 초과 파일
find /path -mtime -7                      # 7일 이내 수정된 파일
find /path -mmin -30                      # 30분 이내 수정된 파일
find /path -empty                         # 빈 파일/디렉토리
find /path -user username                 # 특정 소유자의 파일
find /path -perm 755                      # 특정 권한의 파일
```

### find + exec 조합

> 검색 결과에 대해 바로 명령 실행

```bash
# 찾은 .log 파일 모두 삭제
find /var/log -name "*.log" -mtime +30 -exec rm {} \;

# 찾은 .sh 파일에 실행 권한 부여
find . -name "*.sh" -exec chmod +x {} \;

# 찾은 파일 목록 상세 출력
find . -name "*.conf" -exec ls -l {} \;
```

- `{}` : 검색된 각 파일명이 들어가는 자리
- `\;` : `-exec` 종료 표시
- `+`를 사용하면 파일을 모아서 한 번에 실행 (더 효율적): `-exec ls -l {} +`

## 파일 내용 검색 (grep)

> 파일 내용에서 특정 문자열을 찾는 것. 서버 로그 분석에 필수

```bash
grep "ERROR" app.log                  # 패턴이 포함된 줄 출력
grep -i "error" app.log               # 대소문자 무시
grep -n "ERROR" app.log               # 줄 번호 표시
grep -r "TODO" /src/                  # 디렉토리 재귀 검색
grep -c "ERROR" app.log               # 매칭 줄 수만 출력
grep -v "DEBUG" app.log               # 패턴이 없는 줄 출력 (역매칭)
grep -l "ERROR" *.log                 # 매칭되는 파일명만 출력
grep -A 3 "Exception" app.log        # 매칭 줄 + 뒤 3줄
grep -B 2 "Exception" app.log        # 매칭 줄 + 앞 2줄
grep -C 2 "Exception" app.log        # 매칭 줄 + 앞뒤 2줄
grep -E "ERROR|WARN" app.log         # 정규 표현식 (확장)
```

| 옵션 | 의미 |
|------|------|
| `-i` | 대소문자 무시 |
| `-n` | 줄 번호 표시 |
| `-r` | 디렉토리 재귀 검색 |
| `-v` | 역매칭 (해당 패턴이 없는 줄) |
| `-c` | 매칭 줄 수 |
| `-l` | 파일명만 출력 |
| `-E` | 확장 정규식 사용 |
| `-A/B/C` | 매칭 줄 주변 컨텍스트 출력 |

## 파일 링크 (ln)

> 바로가기를 만드는 것. Windows의 바로가기와 비슷하지만 더 강력

### 심볼릭 링크 (Symbolic Link / Soft Link)

```bash
ln -s /원본/경로 /링크/경로
ln -s /opt/app/config.yml /etc/app/config.yml
```

- 원본 파일의 **경로를 가리키는 포인터**
- 원본이 삭제되면 링크가 깨짐 (dangling link)
- 디렉토리에도 사용 가능
- 서로 다른 파일 시스템 간에도 가능
- 실무에서 가장 많이 사용

### 하드 링크 (Hard Link)

```bash
ln /원본/파일 /링크/파일
```

- 원본과 **같은 inode**(실제 데이터)를 가리킴
- 원본을 삭제해도 링크가 유효 (데이터가 남아있음)
- 같은 파일 시스템 내에서만 가능
- 디렉토리에는 사용 불가

### 비교

```
심볼릭 링크: 링크 → 원본 경로 → 데이터
하드 링크:   원본 ──→ 데이터 ←── 링크   (둘 다 직접 데이터를 가리킴)
```

## 파일 정보 확인

### file - 파일 타입 확인

```bash
file document.pdf        # PDF document, version 1.4
file script.sh           # Bourne-Again shell script, ASCII text executable
file image.png           # PNG image data, 800 x 600, 8-bit/color RGBA
```

### stat - 파일 상세 메타데이터

```bash
stat file.txt
```

```
  File: file.txt
  Size: 4096        Blocks: 8          IO Block: 4096   regular file
Device: 801h/2049d  Inode: 123456      Links: 1
Access: (0644/-rw-r--r--)  Uid: ( 1000/  user)   Gid: ( 1000/  group)
Access: 2026-04-03 10:00:00
Modify: 2026-04-03 09:30:00
Change: 2026-04-03 09:30:00
```

- **Access** : 마지막 읽기 시간
- **Modify** : 마지막 내용 수정 시간
- **Change** : 마지막 메타데이터(권한 등) 변경 시간

### du - 디스크 사용량 확인

> 디렉토리가 얼마나 디스크를 차지하는지 확인

```bash
du -sh /var/log/           # 해당 디렉토리 총 크기
du -sh *                   # 현재 위치의 각 항목 크기
du -sh * | sort -rh        # 크기 순 정렬
du -ah /path --max-depth=1 # 1단계 깊이까지만 크기 표시
```

### wc - 파일 통계

```bash
wc file.txt          # 줄 수, 단어 수, 바이트 수
wc -l file.txt       # 줄 수만
wc -w file.txt       # 단어 수만
wc -c file.txt       # 바이트 수만
```

## 파일 비교 (diff)

> 두 파일의 차이점을 보여주는 것. 코드 리뷰나 설정 변경 확인에 유용

```bash
diff file1.txt file2.txt        # 차이점 출력
diff -u file1.txt file2.txt     # unified 형식 (git diff와 유사)
diff -r dir1/ dir2/             # 디렉토리 간 비교
diff -q file1.txt file2.txt     # 다른지 여부만 출력
```

## 파일 무결성 검증 (md5sum / sha256sum)

> 파일의 "지문"을 찍는 것. 파일 내용이 1바이트라도 다르면 완전히 다른 해시값이 나온다

`diff`가 내용의 **차이점**을 보는 것이라면, 체크섬은 내용이 **동일한지** 확인하는 것이다.

### md5sum

```bash
md5sum file.txt
# 출력: d41d8cd98f00b204e9800998ecf8427e  file.txt

md5sum file1.txt file2.txt    # 여러 파일 해시 한 번에 확인
```

### sha256sum

```bash
sha256sum file.txt
# 출력: e3b0c44298fc1c149afbf4c8996fb924...  file.txt
```

- MD5보다 강력한 해시 알고리즘. 보안이 중요한 경우 사용

### 체크섬 파일로 검증

```bash
# 전송 전: 체크섬 파일 생성
md5sum deploy.tar.gz > checksum.md5

# 전송 후: 수신 측에서 검증
md5sum -c checksum.md5
# deploy.tar.gz: OK
```

### 주요 용도

- **파일 전송 무결성 검증** — 다운로드하거나 서버 간 전송한 뒤, 원본과 해시값이 같은지 비교해서 깨지지 않았는지 확인
- **두 파일이 동일한지 빠르게 비교** — 파일명이 달라도 내용이 같으면 해시값이 동일
- 보안 목적(비밀번호 해싱 등)으로는 MD5가 취약하므로 `sha256sum` 사용

## 파일 압축/해제

### tar - 아카이브 (묶기 + 압축)

> 여러 파일을 하나로 묶는 것. 택배 상자에 여러 물건을 담는 것과 같다

`.tgz`는 `.tar.gz`의 줄임 표현으로, 완전히 동일한 포맷이다. 과거 DOS/Windows에서 확장자가 3글자까지만 허용되던 시절의 관례가 남은 것.

```bash
# 압축
tar -czf archive.tar.gz /path/to/dir    # gzip 압축 (.tgz와 동일)
tar -czf archive.tgz /path/to/dir       # .tar.gz의 줄임 표현
tar -cjf archive.tar.bz2 /path/to/dir   # bzip2 압축 (더 높은 압축률)

# 해제
tar -xzf archive.tar.gz                 # gzip 해제
tar -xzf archive.tar.gz -C /target/     # 특정 디렉토리에 해제
tar -xjf archive.tar.bz2                # bzip2 해제

# 내용 확인 (해제하지 않고)
tar -tzf archive.tar.gz                  # 목록만 보기
```

| 옵션 | 의미 |
|------|------|
| `-c` | create (생성) |
| `-x` | extract (해제) |
| `-z` | gzip 사용 |
| `-j` | bzip2 사용 |
| `-f` | 파일명 지정 |
| `-v` | 처리 과정 출력 |
| `-t` | 목록 확인 |
| `-C` | 해제 대상 디렉토리 지정 |

### zip / unzip

```bash
zip -r archive.zip /path/to/dir    # 디렉토리 압축
zip archive.zip file1 file2        # 여러 파일 압축
unzip archive.zip                  # 해제
unzip archive.zip -d /target/      # 특정 디렉토리에 해제
unzip -l archive.zip               # 내용 목록만 확인
```

### gzip / gunzip

```bash
gzip file.txt              # file.txt → file.txt.gz (원본 삭제됨)
gzip -k file.txt           # 원본 유지하면서 압축
gunzip file.txt.gz         # 해제
gzip -d file.txt.gz        # 해제 (gunzip과 동일)
```

### 안전한 압축 가이드

`tar`와 `zip`은 원본을 **읽기만** 하므로 기존 폴더에 영향을 주지 않는다. 단, `gzip` 단독 사용은 원본을 삭제하므로 주의.

| 명령어 | 원본 유지 여부 |
|--------|--------------|
| `tar -czf` | O (안전) |
| `zip -r` | O (안전) |
| `gzip file` | X (원본 삭제됨) |
| `gzip -k file` | O (`-k` 옵션으로 보존) |

#### 권장 절차

```bash
# 1. 대상 폴더 크기 먼저 확인
du -sh /path/to/folder

# 2. 디스크 여유 공간 확인 (압축 파일이 저장될 공간 필요)
df -h .

# 3. 압축 실행
tar -czf backup_$(date +%Y%m%d).tar.gz /path/to/folder

# 4. 압축 파일 무결성 검증 (해제하지 않고 목록 확인)
tar -tzf backup_20260403.tar.gz | head -20

# 5. (선택) 체크섬 기록
md5sum backup_20260403.tar.gz > backup_20260403.md5
```

#### 특정 파일/폴더 제외하고 압축

```bash
# node_modules, .git 등 제외
tar -czf backup.tar.gz --exclude='.git' --exclude='node_modules' /path/to/project

# 로그 파일 제외
tar -czf backup.tar.gz --exclude='*.log' /path/to/folder
```

## 실무에서 자주 쓰는 조합 예시

```bash
# 30일 이상 된 로그 파일 정리
find /var/log -name "*.log" -mtime +30 -exec rm -f {} \;

# 큰 파일 찾기 (디스크 용량 부족 시)
du -sh /var/* | sort -rh | head -10

# 특정 포트를 사용하는 프로세스의 로그에서 에러 검색
grep -rn "ERROR" /var/log/app/ | tail -20

# 설정 파일 백업 후 수정
cp -p /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak

# 배포 파일 압축 및 전송 준비
tar -czf deploy_$(date +%Y%m%d).tar.gz /opt/app/
```
