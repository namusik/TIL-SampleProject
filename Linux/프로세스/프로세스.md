# 리눅스 프로세스

> 최종 업데이트: 2026-04-22 | 기준: Linux 커널 기준 일반 개념

## 개념

**프로세스(Process)** 는 "**실행 중인 프로그램의 인스턴스**"다. 디스크에 있는 실행 파일(프로그램)이 메모리에 적재되어 CPU 자원을 할당받아 동작하는 상태.

> 비유하자면 **요리 레시피(프로그램)** 를 **실제 주방에서 요리하는 행위(프로세스)**. 같은 레시피(브라우저)를 두 개 실행하면 두 개의 프로세스가 생긴다.

## 프로세스 vs 스레드

| 항목 | 프로세스 | 스레드 |
|------|---------|-------|
| 독립성 | 각자 **독립된 메모리 공간** | **같은 프로세스 안에서 메모리 공유** |
| 생성 비용 | 비쌈 (메모리 복사) | 저렴 |
| 통신 방식 | IPC(파이프, 소켓, 공유 메모리 등) | 변수 공유 |
| 장애 격리 | 하나 죽어도 다른 프로세스 영향 없음 | 하나 죽으면 프로세스 전체 영향 |

## 프로세스의 생명주기

```
fork()      exec()       (실행)        exit()
  │           │             │             │
  ▼           ▼             ▼             ▼
[생성] ──► [프로그램 적재] ──► [Running] ──► [Zombie] ──► [제거]
               │                 │
               │                 ▼
               │             [Sleeping]
               ▼             [Stopped]
            PID 부여
```

- **`fork()`** — 부모 프로세스를 복제해 새 프로세스 생성
- **`exec()`** — 현재 프로세스의 메모리를 다른 프로그램으로 교체
- **`exit()`** — 종료 (자원 반환, 부모에게 상태 전달)
- **Zombie** — 종료는 했지만 부모가 아직 상태를 회수하지 않은 상태

## PID (프로세스 ID)

- **Process ID** — 운영체제가 현재 실행 중인 모든 프로세스에 부여하는 **고유한 정수 식별자**
- 시스템 내 유일, 종료 후 재사용될 수 있음
- 예: `1`번은 항상 **init / systemd** (모든 프로세스의 조상)

### 주요 PID 관련 용어

| 용어 | 의미 |
|------|------|
| **PID** | 프로세스 ID (자신의 ID) |
| **PPID** | Parent PID (부모 프로세스 ID) |
| **PGID** | Process Group ID (프로세스 그룹 ID) |
| **SID** | Session ID (세션 ID — 터미널 단위) |
| **TID** | Thread ID (스레드 ID) |

```sh
echo $$        # 현재 쉘의 PID
echo $PPID     # 현재 쉘의 부모 PID
pidof nginx    # 이름으로 PID 찾기
```

## 프로세스 상태 (STAT)

`ps` / `top` 에서 보이는 상태 코드.

| 코드 | 의미 |
|------|------|
| `R` | Running/Runnable (실행 중 또는 실행 대기) |
| `S` | Sleeping (인터럽트 가능한 대기) |
| `D` | Uninterruptible sleep (I/O 대기 — 보통 중단 불가) |
| `T` | Stopped (정지) |
| `Z` | Zombie (종료됐으나 회수 안 된 상태) |
| `<` | 높은 우선순위 |
| `N` | 낮은 우선순위(nice) |
| `+` | 포그라운드 프로세스 그룹 |

## 파일 디스크립터 (File Descriptor, FD)

- 특정 프로세스가 어떤 파일을 열었을 때, 운영체제가 해당 파일에 부여하는 **간단한 정수 번호**
- 비유: 은행에서 업무를 볼 때 뽑는 **대기 번호표**
  - 프로세스는 파일의 긴 경로(`/var/log/a.log`)를 일일이 사용하는 대신, 운영체제로부터 받은 "3번 창구"나 "5번 번호표" 같은 간단한 FD 번호를 통해 파일을 읽거나 쓴다
- 파일뿐 아니라 **소켓, 파이프, 디바이스**도 FD로 표현 — "Everything is a file"

### 기본 예약 번호

| FD | 용도 |
|----|------|
| `0` | 표준 입력 (stdin, 키보드) |
| `1` | 표준 출력 (stdout, 화면) |
| `2` | 표준 에러 (stderr, 화면) |

프로세스가 새로운 파일을 열 때마다 **`3`, `4`, `5`... 순서로 비어있는 가장 낮은 번호**를 부여받는다.

### FD 확인

```sh
ls -l /proc/<PID>/fd/        # 특정 프로세스가 연 FD 전체
lsof -p <PID>                # 프로세스의 열린 파일 상세
lsof -i :8080                # 특정 포트를 사용 중인 프로세스
```

## 시그널 (Signal)

프로세스에 보내는 **간단한 알림/명령** — 종료·일시정지·설정 재로드 등.

### 자주 쓰는 시그널

| 시그널 | 번호 | 의미 | 특징 |
|-------|-----|------|------|
| `SIGHUP` | 1 | 터미널 종료 / 설정 리로드 신호 | 데몬이 설정 재로드에 관용적으로 사용 |
| `SIGINT` | 2 | 인터럽트 (Ctrl+C) | 사용자 중단 |
| `SIGKILL` | 9 | **강제 종료** | 무시/가로채기 **불가** |
| `SIGTERM` | 15 | **정상 종료 요청** (기본값) | 프로세스가 정리 후 종료 가능 |
| `SIGSTOP` | 19 | 정지 | 무시 불가 |
| `SIGCONT` | 18 | 재개 |

### `kill` 명령

```sh
kill <PID>            # 기본값 SIGTERM (15) — 정상 종료 요청
kill -15 <PID>        # 동일 (명시적 SIGTERM)
kill -9 <PID>         # SIGKILL — 강제 종료. 마지막 수단
kill -HUP <PID>       # 설정 리로드
kill -l               # 모든 시그널 목록

killall nginx         # 이름으로 모두 종료
pkill -f "java.*myapp" # 패턴 매칭으로 종료
```

> **`-9`(SIGKILL)는 최후의 수단**. 프로세스가 정리 작업(파일 닫기, 트랜잭션 커밋)을 할 기회가 없어서 데이터 손상 위험. 먼저 `-15`(SIGTERM)를 시도하고, 안 될 때 `-9`.

## 프로세스 조회 명령

### `ps` — 스냅샷 조회

```sh
ps -ef              # 전체 프로세스 (System V 스타일)
ps aux              # 전체 + CPU/메모리 사용률 (BSD 스타일)
ps -ef | grep java
ps -o pid,ppid,cmd --forest   # 부모-자식 트리
```

자세한 필드 의미는 [ps-command.md](ps-command.md) 참고.

### `top` / `htop` — 실시간 모니터링

```sh
top                 # 기본 프로세스 모니터
top -p <PID>        # 특정 PID만
htop                # 컬러·인터랙티브 개선판 (별도 설치)
```

### `pgrep` / `pidof` — PID 검색

```sh
pgrep nginx
pgrep -f "java.*myapp"
pidof sshd
```

### `/proc/<PID>/` — 커널이 노출하는 가상 파일시스템

```sh
cat /proc/<PID>/status    # 상세 상태
cat /proc/<PID>/cmdline   # 실행 명령줄
ls /proc/<PID>/fd/        # 열린 FD
cat /proc/<PID>/environ   # 환경변수
```

## 포그라운드 / 백그라운드 / Job Control

쉘에서 프로세스를 다루는 방식.

```sh
./server &            # 백그라운드 실행
jobs                  # 현재 쉘의 잡 목록
fg %1                 # 1번 잡을 포그라운드로
bg %1                 # 1번 잡을 백그라운드로
Ctrl+Z                # 포그라운드 → 정지
Ctrl+C                # SIGINT — 중단
nohup ./server &      # 쉘 종료 후에도 살아남음 (SIGHUP 무시)
disown %1             # 쉘 잡 목록에서 제거
```

> 쉘을 나가도 계속 돌아야 할 프로세스는 **`nohup` + `&`** 또는 **`systemd`** 서비스로 등록. 자세한 데몬화는 [linux-daemon.md](linux-daemon.md), [systemd.md](systemd.md) 참고.

## 포트 점유 확인

```sh
lsof -i :8080                    # 8080 포트를 쓰는 프로세스
ss -tulnp | grep 8080            # 같은 기능, netstat 후계자
netstat -tulnp | grep 8080       # (레거시)
fuser 8080/tcp                   # 포트 사용 PID
```

## 자원 사용 제어

| 도구 | 용도 |
|------|------|
| **`nice` / `renice`** | CPU 우선순위 조정 |
| **`ulimit`** | 프로세스별 리소스 한도 (FD, 메모리, CPU 시간 등) |
| **`cgroups`** | 프로세스 그룹 단위의 CPU/메모리 쿼터 (컨테이너의 기반) |
| **`taskset`** | CPU 어피니티 — 특정 코어에 고정 |

```sh
nice -n 10 ./heavy_job    # 낮은 우선순위로 실행
ulimit -n                  # 현재 쉘의 FD 한도
ulimit -n 65535            # FD 한도 변경
```

## 고아/좀비 프로세스

| 유형 | 상태 | 처리 |
|------|------|------|
| **고아(Orphan)** | 부모가 먼저 죽은 자식 | **init/systemd(PID 1)** 이 입양 후 정리 |
| **좀비(Zombie, `Z`)** | 자식이 종료했는데 부모가 `wait()` 안 함 | 부모가 회수하거나 부모 자체를 종료해야 사라짐 |

```sh
ps aux | awk '$8=="Z"'   # 좀비 프로세스 찾기
```

## 백엔드 개발자 관점 실무 포인트

- **JVM 프로세스** — `jps`, `jstack <PID>`, `jmap <PID>` 등 전용 도구와 함께 쓰면 유용
- **컨테이너 안의 PID 1** — 프로세스 정리(reaping) 안 하면 좀비 누적. `--init` 옵션 또는 `tini` 사용
- **SIGTERM 우아한 종료** — 서버 프로세스는 `SIGTERM` 수신 시 **커넥션 드레이닝** 후 종료하도록 설계
- **FD 누수** — `lsof -p <PID>` 로 확인. 파일/소켓 누수 시 `Too many open files`
- **OOM Killer** — 메모리 부족 시 커널이 프로세스를 강제 종료. `/var/log/messages`/`dmesg`에 로그 남음
